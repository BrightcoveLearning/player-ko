플레이어 예제

참고: 이 샘플에 표시된 것과 유사한 기능은 이제 Studio의 일부입니다. 스타일링 플레이어문서. 이 샘플은 로고 위치, 표시 시간 및 기타 옵션에 대한 더 많은 제어를 제공합니다.
이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 이 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

플러그인은 플레이어 상단에 웹 지원 이미지 오버레이를 표시하는 데 사용할 수 있습니다. 다음을 기반으로 메시지 표시를 제어 할 수 있습니다.

  • 위치-이미지가 플레이어의 다른 위치에 표시 될 수 있습니다.
  • 플레이어 이벤트 - 예play : 사용자 지정 이벤트pause
  • 시간 간격 - 비디오 재생 중 지정된 시간

다음 예에서는 플레이어의 왼쪽 하단에 로고가 표시됩니다. 비디오에서 5 초가 지나면 로고가 천천히 사라집니다.

개발 시퀀스

권장되는 개발 시퀀스는 다음과 같습니다.

  1. 인페이지 임베드 플레이어 구현을 사용하여 플레이어, 플러그인 및 CSS의 기능을 테스트합니다 (CSS가 필요한 경우)
  2. 로컬 테스트를 위해 플러그인의 JavaScript와 CSS를 별도의 파일에 넣습니다.
  3. 오류가 발생하면 플러그인 코드와 CSS를 서버에 배포합니다.
  4. Studio를 사용하여 플레이어에 플러그인과 CSS를 추가합니다.
  5. iframe 구현이 더 적합하다고 판단되면 인페이지 임베드 플레이어 구현을 대체합니다 (다음 섹션에서 자세히 설명).

이 단계에 대한 자세한 내용은단계별 을 참조하십시오. 플러그인 개발가이드.

iframe 또는 인페이지 임베드

브라이트코브 플레이어에 대한 향상된 기능을 개발할 때 코드가 iframe 또는 인페이지 임베드 구현에 가장 적합한지 결정해야 합니다. 모범 사례는 iframe 구현과 함께 사용할 플러그인을 빌드하는 것입니다. iframe 플레이어를 사용하면 다음과 같은 이점이 있습니다.

  • 기존 자바스크립트 및/또는 CSS와의 충돌 없음
  • 자동 반응성
  • iframe은 소셜 미디어 앱에서 쉽게 사용할 수 있습니다 (또는 동영상이 다른 앱으로 “여행”해야 할 때마다)

In-Page 임베드 플레이어를 통합하는 것은 더 복잡할 수 있지만, 해당 구현을 중심으로 코드를 계획할 때가 있습니다. 일반화하려면 이 방법은 포함 페이지가 플레이어와 통신해야 할 때 가장 좋습니다. 구체적으로 다음과 같은 몇 가지 예가 있습니다.

  • 포함 페이지의 코드는 플레이어 이벤트를 수신하고 행동해야 합니다.
  • 플레이어는 포함 페이지의 스타일을 사용합니다.
  • iframe은 포함 페이지에서 리디렉션하는 것처럼 앱 로직이 실패합니다.

최종 구현에서 iframe 코드를 사용하지 않더라도 JavaScript용 플러그인과 CSS용 별도의 파일과 함께 인페이지 포함 코드를 사용할 수 있습니다. 이렇게 하면 로직을 캡슐화하여 여러 플레이어에서 쉽게 사용할 수 있습니다.

기타 HTML

그만큼디스플레이 오버레이 플러그인 JavaScript 파일은<script>꼬리표. CSS는디스플레이 오버레이 플러그인페이지에 링크되어 있지 않습니다. 페이지에서 사용자 정의 CSS를 사용하여 쉽게 변경할 수 있습니다.

애플리케이션 흐름

이 응용 프로그램의 기본 로직은 다음과 같습니다.

  • 적절한 시간에 오버레이 이미지를 제거하는 기능이 정의됩니다.
  • 적절한 시간에 오버레이 이미지를 표시하는 기능이 정의됩니다.
  • 사용자 지정 구성 옵션은 기본 옵션과 병합됩니다. 기본 옵션은 사용자 지정 옵션이 제공되지 않은 경우에만 사용됩니다.
  • 오버레이 이미지를 표시하는 함수가 호출됩니다.

애플리케이션 스타일링

길이가 긴 CSS는 오버레이 이미지의 모양과 이미지가 표시되는 위치를 정의합니다. 에 대한 기본 CSS 디스플레이 오버레이 플러그인 CSS 사용자 정의를 쉽게하는 데 사용되지 않습니다.

플러그인 코드

일반적으로 JavaScript를 브라이트코브 플레이어 플러그인으로 변환할 때는 명목상 변경이 한 가지 필수 변경 사항은ready()메서드의 표준 사용을 플러그인을 정의하는 코드로 대체하는 것입니다.

다음은 플레이어와 함께 작동하는 JavaScript 코드에 매우 일반적으로 사용되는 시작입니다.

videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  ...
});

표준 구문을 사용하여 브라이트코브 플레이어 플러그인을 시작하도록 첫 번째 줄을 변경합니다.

videojs.registerPlugin('pluginName', function(options) {
  var myPlayer = this;
  ...
});

앞서 언급했듯이이 문서의 해당 GitHub 저장소에서 플러그인의 JavaScript 코드를 볼 수 있습니다. logo-overlay.js .

플레이어와 함께 플러그인 사용하기

플러그인의 CSS 및 JavaScript 파일을 인터넷에 액세스할 수 있는 위치에 저장하면 플레이어와 함께 플러그인을 사용할 수 있습니다. 스튜디오의 PLAYERS모듈에서 플레이어를 선택한 다음플러그인섹션에서 CSS 및 JavaScript 파일에 URL을 추가하고이름을 추가 할 수 있습니다.및옵션 (옵션이 필요한 경우). 이 플러그인의 경우 옵션은 대부분 다음과 같은 형식으로 플러그인에 전달됩니다.

videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  myPlayer.logoOverlay(
    {
      'align': 'bottom-left',
      'imageURL' : 'https://solutions.brightcove.com/bcls/brightcove-player/logo-overlay/bc-logo.png',
      'clickThruURL': 'https://support.brightcove.com/',
      'start': 'loadstart',
      'end': 5
    }
  )
});
Studio를 사용하여 플레이어에게 플러그인을 할당하는 방법에 대한 자세한 내용은 플레이어 플러그인구성문서를 참조하십시오.
