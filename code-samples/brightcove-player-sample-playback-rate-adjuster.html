플레이어 예제

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 이 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

비디오 재생을 시작하고 컨트롤 바에서 재생 속도 버튼을 선택합니다 (기본값은 1x). 비디오 재생 속도가 어떻게 변하는지 확인하려면 다른 속도를 선택하십시오. 재생 속도를 변경하면 표시되는 지속 시간이 선택한 속도에 따라 다시 계산됩니다.

펜보기재생 속도 조절기게시자 : Brightcove Learning Services ( 쫑끼리 ) 의 위에 CodePen .

코드펜 사용하기

위의 CodePen을 효과적으로 사용하기 위한 몇 가지 팁은 다음과 같습니다.

  • 결과버튼을 클릭하여 플레이어의 실제 표시를 토글합니다.
  • HTML/CSS/JS버튼을 클릭하여 코드 유형 중 하나를 표시합니다.
  • 이 문서의 뒷부분에서 응용 프로그램에서 사용되는 로직, 흐름 및 스타일링은플레이어/HTML 구성 , 응용프로그램 흐름및응용 프로그램 스타일링에서 설명합니다.섹션. 이러한 섹션의 정보와 함께 수행하는 가장 좋은 방법은 다음과 같습니다.
    1. CodePen에서 CODEPEN에 있는편집버튼을 클릭하고 하나의 브라우저/브라우저 탭에서 코드를 사용할 수 있도록 합니다.
    2. CodePen에서 표시할 코드를 조정합니다. CodePen 내에서 서로 다른 코드 섹션의 너비를 변경할 수 있습니다.
    3. 보기플레이어/HTML 구성 , 신청 흐름및/또는애플리케이션 스타일링다른 브라우저/브라우저 탭의 섹션. 이제 코드 설명을 따르고 동시에 코드를 볼 수 있습니다.

개발 시퀀스

권장되는 개발 시퀀스는 다음과 같습니다.

  1. 인페이지 임베드 플레이어 구현을 사용하여 플레이어, 플러그인 및 CSS의 기능을 테스트합니다 (CSS가 필요한 경우)
  2. 로컬 테스트를 위해 플러그인의 JavaScript와 CSS를 별도의 파일에 넣습니다.
  3. 오류가 발생하면 플러그인 코드와 CSS를 서버에 배포합니다.
  4. Studio를 사용하여 플레이어에 플러그인과 CSS를 추가합니다.
  5. iframe 구현이 더 적합하다고 판단되면 인페이지 임베드 플레이어 구현을 대체합니다 (다음 섹션에서 자세히 설명).

이 단계에 대한 자세한 내용은단계별 을 참조하십시오. 플러그인 개발가이드.

iframe 또는 인페이지 임베드

브라이트코브 플레이어에 대한 향상된 기능을 개발할 때 코드가 iframe 또는 인페이지 임베드 구현에 가장 적합한지 결정해야 합니다. 모범 사례는 iframe 구현과 함께 사용할 플러그인을 빌드하는 것입니다. iframe 플레이어를 사용하면 다음과 같은 이점이 있습니다.

  • 기존 자바스크립트 및/또는 CSS와의 충돌 없음
  • 자동 반응성
  • iframe은 소셜 미디어 앱에서 쉽게 사용할 수 있습니다 (또는 동영상이 다른 앱으로 “여행”해야 할 때마다)

In-Page 임베드 플레이어를 통합하는 것은 더 복잡할 수 있지만, 해당 구현을 중심으로 코드를 계획할 때가 있습니다. 일반화하려면 이 방법은 포함 페이지가 플레이어와 통신해야 할 때 가장 좋습니다. 구체적으로 다음과 같은 몇 가지 예가 있습니다.

  • 포함 페이지의 코드는 플레이어 이벤트를 수신하고 행동해야 합니다.
  • 플레이어는 포함 페이지의 스타일을 사용합니다.
  • iframe은 포함 페이지에서 리디렉션하는 것처럼 앱 로직이 실패합니다.

최종 구현에서 iframe 코드를 사용하지 않더라도 JavaScript용 플러그인과 CSS용 별도의 파일과 함께 인페이지 포함 코드를 사용할 수 있습니다. 이렇게 하면 로직을 캡슐화하여 여러 플레이어에서 쉽게 사용할 수 있습니다.

미들웨어 함수

이 샘플은 Video.js 미들웨어를 사용하여 코드 솔루션을 구현합니다. 매우 높은 수준에서 미들웨어가 가장 중요한 사용 사례는 소스 또는 현재 시간 설정과 같이 플레이어의 핵심 동작을 가로채고 변경하려는 경우입니다. 이 문서에서는 일반적으로 Video.js 미들웨어에 대해서는 설명하지 않지만 미들웨어 함수에 사용되는 실제 코드는 설명되어 있습니다. 다음은 미들웨어를 자세히 설명하는 Video.js 소스의 문서입니다.

  • 미들웨어 (Video.js 설명서 세트)
  • 기능 스포트라이트: 미들웨어 (Video.js 블로그에서)
  • 비디오-재생 속도 조정기 (샘플 앱)

브라이트코브 플레이어로 미들웨어 기능 구현

미들웨어 함수를 사용하는 가장 간단한 방법은 다음과 같습니다.

  1. 미들웨어 함수를 작성합니다.
  2. 플레이어가script태그에 있는 페이지에 미들웨어 함수를 배치하거나src속성을 사용하여 미들웨어 함수를 포함시킵니다. script태그. 두 구현 모두 이 문서에 나와 있습니다.
  3. 다음과 같이use()메서드를 사용하여 플레이어에 미들웨어 함수를 등록합니다. 
    videojs.use('mimeType', functionName);

    를 위해마임타입다음과 같은 표준 비디오 MIME 유형을 사용할 수 있습니다. 비디오/mp4 , 또는 사용 *미들웨어 기능이 모든 mimetype에 작용하기를 원하는 경우.

    플러그인과 미들웨어는 스크립트가 포함될 때 자동으로 등록되도록 설정할 수 있습니다. 이 경우에는 다음을 사용할 필요가 없습니다. videojs.use()방법.

미들웨어 함수 정의

레이블이 지정된 코드를 찾습니다.

// +++ Define the middleware functions +++

다음 미들웨어 기능을 작성하십시오.

  1. 선택한 재생 속도에 따라 재생 시작 및 종료 시간을 다시 계산하는 지원 미들웨어 기능을 만듭니다.

  2. 검색 할 때 시작 시간, 기간 및 현재 시간까지 업데이트하는 미들웨어 기능을 만듭니다.

    미들웨어 함수를 만드는 기본 구문은 다음과 같습니다.

    const playbackrateAdjuster = function(player) {
   return {
   ...
 };

    player객체가 함수에 전달된다는 점에 유의하십시오.

    이 미들웨어 기능은 다음을 수행합니다.

    1. ratechange 이벤트 수신 : 재생 속도가 변경되면durationchange과timeupdate이벤트.

    2. setSource () 메서드 구현 : 이것은 모든 미들웨어 함수에 필요한 방법입니다. 표시된 구문을 사용하면 비디오 유형에 관계없이 미들웨어가 항상 선택된다는 것을 의미합니다. 
      setSource(srcObj, next) {
  next(null, srcObj);
},

    3. 현재 재생 속도를 사용하여 메서드 재정의 : 에 대한 메서드 재정의duration , currentTime , buffered , seekable과played현재 재생 속도를 사용하여 계산 된 값으로.

별도의 JavaScript 파일의 코드

다음 코드에서 미들웨어 기능에 대한 JavaScript와use()등록 방법은 페이지에 포함되어 있습니다.

<video-js id="myPlayerID"
	  data-video-id="5604123528001"
	  data-account="1752604059001"
	  data-player="default"
	  data-embed="default"
	  data-application-id
	  class="video-js"
	  width="640" height="360"
	  controls muted></video>
	<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

  <!-- Include the rate adjuster middleware functions -->
  <script src="rate-adjuster.js"></script>

  <script type="text/javascript">
  	videojs.getPlayer("myPlayerID").ready(function(){
  		// When the player is ready, get a reference to it
  		var myPlayer = this;
  		console.log("myPlayer",myPlayer)
  		// Define the playback rate options
  		options = {"playbackRates":[0.5, 1, 1.5, 2, 4]};
  		// Initialize the playback rate button
  		myPlayer.controlBar.playbackRateMenuButton = myPlayer.controlBar.addChild('PlaybackRateMenuButton', {
      	playbackRates: options.playbackRates
  	  });
    });
  </script>

이 예에서는 사용자가 자체 서버에서 JavaScript 파일을 호스팅한다고 가정합니다. 전체 코드는플러그인 섹션 GitHub 저장소에서.
