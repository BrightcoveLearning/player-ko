플레이어 모양 사용자 지정

이 항목은 Brightcove Player의 모양 사용자 지정에 대한 입문서입니다. 콘텐츠는 Chrome의 개발 도구를 사용하지만 모든 최신 브라우저에는 동일한 기능이 있습니다. 여러 브라우저에서 개발 도구를 사용하는 방법에 대한 간략한 소개는디버깅 기초문서.
이 문서는 CSS를 사용하여 플레이어를 사용자 정의합니다. Studio에서 사용할 수있는 제한된 맞춤 설정 옵션이 있으며 구현하는 데 CSS에 대한 지식이 필요하지 않습니다. 참조스타일링 선수자세한 내용은 문서를 참조하십시오.

자체 스킨 만들기

기본 스킨을 사용하지 않고 자신만의 스킨을 생성하려는 경우 플레이어 관리 API를 사용하여 이 작업을 수행할 수 있습니다. 플레이어 구성 가이드는 필요한 정보를제공합니다.

플레이어

참고: 이 문서에서는 CSS에 대한 지식과 브라우저의 개발 도구를 사용하여 HTML 요소의 클래스를 확인하고 적용 가능한 CSS를 검토합니다. 필수 기본 지식이없는 경우 다음을 통해 작업하는 것이 가장 좋습니다. 단계별 : 플레이어 커스터마이징 .

플레이어의 모양을 변경할 수 있지만, 먼저 플레이어 모양을 해결할 방법이 있어야 합니다. 이를 보려면 고급 (인 페이지 임베드) 코드가 삽입 된 HTML 페이지를 찾아보십시오. 에서집단개발 도구 섹션에서<video-js>태그에 값이 있는 클래스가 있는 것을 볼 수 있습니다. video-js많은 다른 사람들 사이에서 할당되었습니다.

이 사실을 알고 나면 스타일을 사용하여 플레이어 자체를 변경할 수 있습니다. 예를 들어 플레이어 주위에 테두리를 포함하려면 다음 스타일을 사용할 수 있습니다.

<style>
  .video-js {
    border-style: double;
    border-width: thick;
    border-color: red;
  }
</style>

이프레임 플레이어

플레이어의 표준 (iframe) 구현을 사용하는 경우 상황이 다릅니다. 여전히 플레이어가video-js클래스이지만, 물론 iframe 내부와video-js꼬리표.

만드는 스타일은 iframe에서 플레이어와 계속 작동하지만, CSS 파일을 만들고 Studio를 사용하여 플레이어와 연결해야 합니다. 당신은플레이어모듈을 클릭 한 다음 CSS를 연결하려는 플레이어를 클릭 한 다음플러그인> 스타일 시트섹션에 CSS 파일 경로를 추가하십시오.

iframe 자체를 맞춤 설정하려면 요소 선택기를 사용하면됩니다. 그런 다음<style>태그를 사용하여 원하는 대로 수정합니다. 아래 예에서는 플레이어 주위에 테두리가 추가됩니다.

<!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js {
      border-style: double;
      border-width: thick;
      border-color: red;
    }
  </style>
</head>

<body>

<iframe src="https://players.brightcove.net/1507807800001/default_default/index.html?videoId=6065136830001"
  allowfullscreen=""
  allow="encrypted-media"
  width="640" height="360"></iframe>

</body>

</html>

재생 버튼

재생 버튼의 모양을 변경하려면 먼저 해결 방법을 알아야합니다. Chrome에서 버튼을 마우스 오른쪽 버튼으로 클릭합니다. 표시되는 선택 항목에서검사 . 선택할 때검사 , 크롬의 개발 도구열릴 것이다.

에서집단개발 도구 섹션에서 버튼 요소에 해당하는 HTML 코드를 볼 수 있습니다.

위의 강조 표시된 코드에서 버튼에 할당된 클래스가 인 것을 볼 수vjs-big-play-button있습니다. 재생 버튼의 색상 (호랑이와 일치) 과 불투명도는 다음과 같은 스타일로 변경됩니다.

.vjs-big-play-button {
  background-color: #B37D5B;
  opacity: .6;
}

이 접근 방식은 효과가 있다고 생각하지만 그렇지 않습니다. CSS로 작업할 때는 속성 특수성을 알고 있어야 합니다. 이는 관련성에 따라 요소에 적용할 속성 값을 브라우저가 결정하는 방법을 나타냅니다.

버튼 변경 사항을 적용하는 한 가지 방법은 선택기에서 다른 클래스를 추가하는 것입니다. 예를 들어, .video-js선택 자의 특이성을 높이기 위해 클래스. 자세한 내용은 CSS 특이성부분

그런 다음 CSS에서 플레이어의 큰 재생 버튼을 다음과 같이 참조합니다.

.video-js .vjs-big-play-button {
  background-color: #B37D5B;
  opacity: .6;
}

버튼을 진한 회색에서 주황색으로 변경한 결과는 다음과 같습니다.

호버 텍스트 변경

큰 재생 버튼 위로 마우스를 가져갈 때 표시되는 텍스트를 변경하려면동영상 재생 기본적으로 동영상에 더 구체적인 항목을 지정할 수 있습니다. 원하는 설명을 인수로 사용하여 다음 JavaScript를 추가하십시오. controlText방법.

가시성 제어

컨트롤 막대와 그 안에 포함 된 컨트롤을 표시할지 여부를 제어 할 수 있습니다. 플레이어 생성을 시작할 때 컨트롤 막대를 숨기려면 다음 스타일을 사용할 수 있습니다.

.video-js .vjs-control-bar {
  display: none;
}

일부 이벤트에 따라 동적으로이 작업을 수행하려면 다음을 사용할 수 있습니다. controlBar.hide()/show()행동 양식. 아래 코드 스 니펫은 메소드 사용을 보여줍니다 (이는id값으로myPlayer비디오) :

<video-js id="myPlayerID"

	class="video-js"

<script src="https://players.brightcove.net/1507807800001/default_default/index.min.js"></script>

<p><button onclick="hideBtn()">hide controls</button></p>
<p><button onclick="showBtn()">show controls</button></p>

컨트롤은 숨기지 않습니다

컨트롤이 숨겨지지 않도록 만들 수도 있습니다. 이렇게하려면transform.none스타일. 선택자는 변환의 특이성을 위해 6 개의 클래스가 사용되는 한 매우 길다.

.video-js.not-hover.vjs-has-started.vjs-playing.vjs-user-active .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible),
.video-js.not-hover.vjs-has-started.vjs-playing.vjs-user-inactive .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible),
.video-js.vjs-has-started.vjs-paused.vjs-ad-playing.vjs-user-inactive .vjs-control-bar,
.video-js.vjs-has-started.vjs-playing.vjs-user-inactive .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible){

약간의 해킹으로 간주되지만 컨트롤 바의 클래스를 8번 반복하여 위의 선택기와 일치하도록 특수성을 구축할 수 있습니다. 
.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar {
  transform: none;
  -webkit-transform: none;

이 CSS를 사용하여 컨트롤바가 사라지는 것을 막는 것은 컨트롤바의 요소를 마우스 오른쪽 버튼으로 클릭하여 검사할 때 매우 유용합니다. 이 작업을 수행하지 않으면 마우스 오른쪽 버튼을 클릭하면 컨트롤 막대가 사라집니다. 또한 첫 번째 마우스 오른쪽 버튼을 클릭하면 Brightcove Player 정보 메뉴가 표시되고두 번째 마우스 오른쪽 버튼을 클릭하면 검사를 선택할 수 있는 메뉴가 표시됩니다. .

글꼴

제목과 설명은 플레이어가 처음로드 될 때와 사용자가 플레이어를 가리킬 때 나타납니다. 제목과 설명의 글꼴 스타일을 함께 또는 개별적으로 변경할 수 있습니다. 에 대한 HTML 독다음 스크린 샷과 같이 나타납니다. 거기에 유의하십시오<div>클래스가있는 요소vjs-dock-text다른 두 개를 포함하는<div>포함하는 요소표제과기술 .

스타일을 설정하여 제목과 설명 모두에 영향을 미칠 수 있습니다. vjs-dock-text수업. 스타일은.vjs-dock-title과.vjs-dock-description물론 더 높은 수준에서 설정된 스타일에 의해 완전히 무시되지는 않습니다. 다음 스타일과 그 결과가 텍스트에 표시되는 것처럼 일부 스타일은 상속됩니다.

.video-js .vjs-dock-text {
  color: yellow;
  font-size: .7em;
}

개별 제목과 설명의 스타일을 직접 지정할 수 있습니다. 다음은에 설정된 스타일을 보여줍니다. .vjs-dock-title과.vjs-dock-description스타일을 표시하는 결과 텍스트와 함께 클래스 선택기 :

.video-js .vjs-dock-title {
  color: red;
  font-size: 1.5em;
}
.video-js .vjs-dock-description {
  color: yellow;
  font-size: 1.0em;
}

컨트롤 바 아이콘 조작

HTML 순서에 따라 결정되는 컨트롤 바 아이콘 순서<div>부모 컨트롤 바의 s <div> . 여기에 컨트롤 바가 있습니다. <div>모든 자녀와 함께.

일반적으로 컨트롤 바의 아이콘에 배치 할 때 사용하는 방법은 다음과 같습니다.

  • 컨트롤 바에 삽입하려는 요소를 동적으로 빌드합니다.
  • controlbar 요소에 대한 참조를 가져옵니다.
  • 컨트롤 막대 앞에 새 요소를 삽입하려는 요소에 대한 참조를 가져옵니다.
  • JavaScript 사용insertBefore()아이콘을 배치하는 방법.

구문insertBefore() is :

parent.insertBefore(newElement,insertBeforeElement)

즉, 아래의 자세한 코드에서 컨트롤 막대 (부모)에 대한 참조를 가져 오는 JavaScript를 볼 수 있습니다. <div>컨트롤 막대에 삽입 할 아이콘과 마지막으로 새 아이콘이 삽입 될 요소에 대한 참조가 있습니다.

플레이어가 처음로드되고 플레이어 위로 마우스를 가져 가면 비디오 제목이 표시되고 플레이어 상단 부분에 검은 색에서 투명한 그라디언트가 표시됩니다. 이 섹션에서는 그라디언트를 변경하는 방법을 보여줍니다.
