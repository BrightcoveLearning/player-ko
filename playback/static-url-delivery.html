소개

Brightcove는 Video Cloud 라이브러리의 자산에 대한 비디오 매니페스트 파일에 대한 정적 URL을 제공합니다. 이를 통해 자체 CMS에서 콘텐츠를 관리하고 사용자 지정 보안 스키마를 사용하여 콘텐츠를 제공할 수 있는 유연성을 얻을 수 있습니다.

이는 매니페스트 URL이 필요하기 전에 Playback API 호출을 허용하지 않는 기존 아키텍처가 있는 고객에게 중요합니다. 플레이어는 이 기능을 사용하여 한 번의 호출을 제거하여 재생 시작 시간을 줄일 수도 있습니다.

또한 참조재생 API v2 참조 .

요구 사항

정적 URL에는 다음 요구 사항이 필요합니다.

  • 브라이트코브 재생 API v2
  • Dynamic Delivery를 위해 자산이 수집됩니다.

애널리틱스 등록

정적 URL을 사용하면 Brightcove Player를 사용하지 않고 Playback API와 통합됩니다. Brightcove Player가 분석 데이터를 자동으로 Brightcove로 보내는 동안 데이터 수집 API를 사용하여 이를 수행합니다.

Data Collection API에 익숙하지 않은 경우 다음을 검토하십시오. 개요: 데이터 수집 API문서.

데이터 수집기로 정보를 보내는 방법에 대한 자세한 내용은 다음을 참조하십시오. 데이터 수집 API 샘플 .

비디오 클라우드

오늘날 VideoCloud를 통한 모든 재생에는 비디오 재생을 시작하기 위해 2개의 API 호출이 필요합니다. 정적 URL을 사용하면 API 호출 수를 하나만으로 줄일 수 있습니다.

다음 다이어그램은 정적 URL 없이 비디오 재생을 위한 API 호출을 보여줍니다.

정적 URL 구현

정적 URL에는 다음이 포함됩니다.

  • 계정 ID
  • 동영상 ID(또는 참조)
  • 전달 유형(HLS, Dash)
  • 구성 옵션(SSAI, DRM, CDN 등)

정적 URL은 고객 생성 및 서명을 사용하여 보호됩니다. JSON 웹 토큰 (JWT). 이 토큰을 사용하여 전달 측면을 구성하고 시행할 수 있습니다. 자세한 내용은만들기 JSON 웹 토큰부분.

도메인

정적 URL에 사용되는 도메인은 Playback API v2(PAPIv2) URL입니다. 

https://playback.brightcovecdn.com/playback/v1/

끝점

각 전송 유형에 대한 정적 URL 예:

배송 유형 URL
HLS 마스터 매니페스트 
/accounts/{account_id}/videos/{video_id}/master.m3u8?bcov_auth={jwt}
대시 매니페스트 
/accounts/{account_id}/videos/{video_id}/manifest.mpd?bcov_auth={jwt}
HLS VMAP[1-1] 
/accounts/{account_id}/videos/{video_id}/hls.vmap?bcov_auth={jwt}
대시 VMAP[1-1] 
/accounts/{account_id}/videos/{video_id}/dash.vmap?bcov_auth={jwt}
최고 비트레이트 MP4 
/accounts/{account_id}/videos/{video_id}/high.mp4?bcov_auth={jwt}
최저 비트 전송률 MP4 
/accounts/{account_id}/videos/{video_id}/low.mp4?bcov_auth={jwt}
썸네일 이미지 
/accounts/{account_id}/videos/{video_id}/thumbnail.jpg?bcov_auth={jwt}
포스터 이미지 
/accounts/{account_id}/videos/{video_id}/poster.jpg?bcov_auth={jwt}

만들기 JSON 웹 토큰

Brightcove의 Playback API와 통신할 때 사용할 토큰을 만들려면만들기 JSON 웹 토큰 (JWT)문서. 여기에서 정적 URL 전달에 사용할 수 있는 클레임을 찾을 수 있습니다.

콘텐츠 보호(DRM) 사용

다음을 포함할 수 있습니다. pro보호 유형을 지정하기 위해 위에서 정의한 것처럼 클레임.

라이선스 요청의 경우 JWT에는 페이로드가 필요하지 않지만 매니페스트 요청 JWT와 동일한 RSA 키를 사용하여 서명해야 합니다.

Widevine 라이선스 요청

다음 엔드포인트를 사용하여 Widevine에 대한 DRM 라이선스를 얻을 수 있습니다. 

/accounts/{account_id}/videos/{video_id}/licenses/cenc/widevine?bcov_auth={JWT}

FairPlay 앱 인증서

FairPlay 앱 인증서의 경우 엔드포인트는 다음과 같습니다. 

/accounts/{account_id}/fairplay_app_cert

PlayReady 엔드포인트

PlayReady의 경우 라이선스 요청에 대한 엔드포인트는 다음과 같습니다.

/accounts/{account_id}/videos/{video_id}/licenses/cenc/playready?bcov_auth={JWT}

노트

다음 참고 사항은 정적 URL 배달에 적용됩니다.

  • 매니페스트 내 캡션이 지원됩니다. 대역 외 사이드카 캡션은 향후 릴리스에서 지원될 예정입니다.
  • 고객의 BYO CDN은 비디오 세그먼트를 제공하는 반면 Brightcove의 CDN은 계속해서 매니페스트 및 하위 매니페스트를 제공합니다.
  • Brightcove는 만료되지 않는 JWT 사용을 지원합니다. 이렇게 하려면 제외exp토큰을 생성할 때 주장하십시오.
