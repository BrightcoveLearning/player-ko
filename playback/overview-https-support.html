개요: HTTPS 지원

{{ page.description }}

HTTPS를 지원하는 이유는 무엇입니까?

HTTPS 지원이 필요한 이유에 대해 먼저 이야기하겠습니다.

  • 소셜 공유 사이트에서 필요합니다. Twitter 및 Facebook은 비디오 플레이어를 표시하기 위해 HTTPS 지원 수준이 필요합니다.
  • 브라우저 잠금을 그대로 유지합니다. 사이트 전체 또는 일부가 HTTPS에있는 경우 "브라우저 잠금"을 그대로 유지하는 것이 좋습니다. 완전한 HTTPS 지원이 없으면 브라우저 잠금이 깨져 보입니다.
  • 플레이어가 계속 작동하도록합니다. 점점 더 많은 브라우저가 HTTPS를 사용하는 사이트에서 HTTP 콘텐츠로드를 거부합니다. 따라서 비디오 플레이어가 HTTPS를 지원하지 않으면로드되지 않거나 기능이 누락됩니다. 도메인 제한이 사용 중일 때 작업중인 플레이어가있는 것은 현재 플레이어의 특정 문제이기도합니다.

HTTPS 활성화

Brightcove Player에서 HTTPS를 활성화하려면 다음을 수행해야합니다.

플레이어에 HTTPS 사용

  • iframe 구현의 경우 페이지가 HTTPS를 통해로드되는지 확인하세요. 아래 예에서 플레이어의src속성은 프로토콜 상대 URL을 사용합니다. 이는 HTTPS 페이지에 삽입 된 플레이어가 HTTPS를 통해로드됨을 의미합니다. 
    <iframe src='//players.brightcove.net/1752604059001/default_default/index.html?videoId=4029697544001' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
  • 인 페이지 구현의 경우 페이지가 HTTPS를 통해로드되는지 확인하세요. 아래 예에서 플레이어의 JavaScript 파일에는src프로토콜 상대 URL을 사용하는 속성. 따라서 HTTPS 페이지에 삽입 된 플레이어는 HTTPS를 통해로드됩니다. 
    <video-js
  data-video-id="4029697544001"
  data-account="1752604059001"
  data-player="default"
  data-embed="default"
  class="video-js" controls></video-js>
<script src="//players.brightcove.net/1752604059001/default_default/index.min.js"></script>
물론, 당신은 사용할 수 있습니다 https :상대 URL을 사용하는 대신 플레이어 요청에서 이것이 대부분의 문서에서 수행되는 작업임을 많은 사람들이 알고 있습니다.

HTTPS를 통해 콘텐츠 제공

모든 VOD 콘텐츠가 HTTPS를 통해 제공되는지 확인하세요.

  • Video Cloud 계정의 경우 모든 콘텐츠가 HTTPS를 통해 자동으로 제공됩니다.
  • BYO 계정의 경우 공급자에게 연락하여 고객 지원에 후속 조치를 취해야 할 수 있습니다.

HTTPS를 통해 라이브 스트림 제공

원격 자산 URL이 HTTPS를 사용하고 있는지 확인하십시오. 예를 들면 다음과 같습니다.

"https://livedelivery-lh.akamaihd.net/i/mylivestream_1@146180/master.m3u8"
