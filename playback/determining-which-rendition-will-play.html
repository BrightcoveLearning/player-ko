HLS 또는 MP4?

새로운 Brightcove Player를 사용할 때 가장 많은 수의 장치와 브라우저에 도달하기 위해 HLS 및 MP4 변환이 모두있는 수집 프로필을 선택하거나 생성하는 것이 좋습니다. 이 표는 플레이어가 런타임에 특정 브라우저에서 재생할 렌 디션을 결정하는 방법을 보여줍니다.

브라우저 유형 기본적으로 사용되는 재생 기술
Android 5.0 이상에서 Desktop Chrome 34 이상, Desktop Firefox 42 이상, Edge 및 Chrome 34 이상 HTML 기반 HLS (MSE 사용)
데스크탑 (macOS) Safari, 모바일 (iOS) Safari 네이티브 HLS (브라우저에서 구현)
이전 Firefox 및 Chrome 플래시 기반 HLS; Flash 10.3+ 필요 (Flash가 비활성화 된 경우 MP4로 대체 됨)
Windows 8.1 이상에서 IE 11 HTML 기반 HLS (MSE 사용)

HLS 및 DASH 변환 선택

HLS과대시비디오가 세그먼트로 나뉩니다. 일반적으로 약 10 초 정도 소요되지만 더 길거나 짧을 수 있습니다. 대역폭과 해상도를 알고있는 경우 플레이어는 이러한 기준에 따라 변환을 선택합니다. 해상도 또는 대역폭을 알 수없는 경우 (예 : display:none ) 플레이어는 .5 MB / s (4000kbs에 해당)에 가장 가까운 변환으로 시작합니다. 세그먼트 경계에서는 아래 텍스트와 이미지에 설명 된 더 높거나 낮은 변환으로 전환됩니다.

HLS와 DASH는 모두 사용 가능한 대역폭과 인코딩을 고려할 때 가능한 최고 품질의 시청 경험을 보장하고 동시에 플레이어 크기를 고려합니다. 이것이 항상 사용 가능한 가장 높은 비트레이트 변환을 사용하는 것을 의미하지는 않습니다. 예를 들어 플레이어가 300x150 픽셀이면 4k 스트림을 다운로드하는 데 대역폭이 낭비됩니다.

기본적으로 플레이어는 가장 최근에 감지된 세그먼트 대역폭보다 작은 가장 높은 비트레이트 배리언트를 로드하려고 합니다. 한 가지 조건으로, 현재 플레이어 크기보다 큰 배리언트가 여러 배리언트가 있는 경우 현재 플레이어 크기보다 한 크기만 전환됩니다.

재생하는 동안 플레이어는 다음 알고리즘에 따라 더 높거나 낮은 변환으로 전환됩니다. 이 알고리즘에 대한 입력은 다음과 같습니다.

CSS를 사용하여 플레이어를 숨기는 경우(예: display: none ) 비디오를 재생할 준비가 될 때까지 플레이어는 재생 디스플레이 크기를 0px X 0px로 결정하고 처음에 재생할 가장 낮은 품질의 변환을 선택합니다. 이를 방지하려면 플레이어를 숨기거나 비디오를 재생할 준비가 되었을 때 동적으로 로드하는 다른 방법을 찾으십시오.
  • 사용 가능한 대역폭
  • 플레이어 차원

고급 알고리즘 개요

  1. 측정 된 대역폭보다 비트 전송률이 높은 모든 변환을 제거합니다.
  2. 해상도 (가로줄 수)를 기준으로 나머지 변환을 높음에서 낮음으로 정렬합니다.
  3. 플레이어 차원에 가장 가까운 것을 가리 킵니다.
  4. 그보다 하나 더 높은 것을 선택하십시오.

프로세스는 다음과 같습니다.

  1. 새 세그먼트가 다운로드 될 때마다 다운로드 비트 전송률은 세그먼트의 크기와 다운로드에 걸린 시간을 기준으로 계산됩니다.
  2. 새 측정 값보다 비트 전송률이 더 높은 모든 변환이 필터링됩니다.
  3. 현재 플레이어의 크기보다 큰 모든 변환은 필터링됩니다.
  4. 플레이어가 하나의 픽셀이 너무 작기 때문에 품질이 크게 저하되는 것은 원하지 않으므로 다음으로 높은 해상도로 다시 추가합니다. 남아있는 가장 높은 비트 전송률 변환은 사용되는 변환입니다.

위에서 설명한 필터링에 따라 허용되는 변환이없는 것으로 판명되면 마스터 재생 목록에 나열된 첫 번째 인코딩이 사용됩니다.

플레이어가 다른 우선 순위 집합을 사용하도록하려면 렌 디션 선택 논리를 완전히 바꿀 수 있습니다. 예를 들어 재생 중에 더 많은 지연이 발생할 수 있지만 항상 해상도별로 가장 적절한 렌 디션을 선택할 수 있습니다. 에 대한 설명서를 참조하십시오. player.hls.selectPlaylist자세한 사항은.

MP4 변환 선택

모바일 장치에서 실행하고 MP4를 재생하는 경우 (위에 나열된 규칙에 따라) 플레이어는 .5 MB / s에 가장 가까운 비트 전송률을 가진 MP4를 선택합니다. 데스크톱 또는 랩톱 장치의 경우 3MB / s에 가장 가까운 장치를 선택합니다.

노트

  • HLS는 demuxed 오디오 시나리오에서 오디오 스트림을 조정하는 것을 허용하지 않습니다. 따라서 위의 예와 같이 다른 오디오 그룹이나 대체 오디오가있는 경우 스테레오 또는 서라운드 트랙의 여러 품질 변환을 지정할 수 없습니다.

    그러나 재생 목록이 오디오 전용 인 경우 일반 오디오를 통해 제공되는 경우 가능합니다. EXT-X-STREAM-INF꼬리표.

  • DASH의 경우 적응 형 오디오를 사용할 수 있지만 오디오 전용 매니페스트가 아닌 경우 현재 적응 형 오디오를 지원하지 않습니다. 비디오 변환과 관련된 단일 변환을 선택합니다. 이것이 우리가 찾은 최초의 오디오 렌 디션입니다.
