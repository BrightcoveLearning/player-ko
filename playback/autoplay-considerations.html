자동 재생 고려 사항

{{ page.description }}

개요

페이지가로드 될 때 Brightcove Player를 사용하여 비디오를 자동 재생하는 것은 간단한 문제가 아닙니다. 이것은 비디오 자동 재생을 원하지 않는 사용자의 희망에 응답하는 브라우저 공급 업체에서 기인합니다. 이 문서는 자동 재생에 영향을 미치는 다양한 브라우저 및 Brightcove Player 구성에서 비디오 자동 재생의 현재 상태를 자세히 설명합니다.

우리는 세트가 있습니다사용 가능한 테스트 케이스 .

결론

세부 사항없이 최선의 조언을 원한다면 :

클라이언트 측 광고없이 자동 재생

다음 단계는 클라이언트 측 광고가 없거나 SSAI가있는 플레이어에서 성공적인 자동 재생을위한 최상의 기회를 제공합니다.

  • 세트autoplay ...에"any"
  • 세트playsinline에게true (아이폰용)

클라이언트 측 광고로 자동 재생

다음 단계는 클라이언트 측 광고 (IMA3 또는 FreeWheel)가있는 플레이어에서 성공적인 자동 재생을위한 최상의 기회를 제공합니다.

  • 세트autoplay ...에true
  • 세트playsinline에게true (아이폰용)

자동 실행 구성

Brightcove Player는 자동 재생에 대해 5 가지 가능한 값을 지원합니다. HTML5 사양에서 자동 재생은 부울 속성입니다. true ) 또는 꺼짐 ( false ). Brightcove Player에는 세 가지 특수 값을 사용할 수 있습니다.

  • play : 그러면 Brightcove Player가play()소스가 설정되었을 때.
  • muted : 그러면 Brightcove Player가 플레이어를 음소거하고play() .
  • any : 그러면 Brightcove Player가play() . 실패하면 플레이어를 음소거하고play()다시.

이 세 가지 값은 태그 속성에서 지원되지 않습니다!

우선순위

다음 목록은 자동 재생을 구성 할 수있는 모든 방법과 Brightcove Player의 초기화 프로세스에서 우선 순위가 지정되는 방법을 설명합니다.

  1. 비디오 요소 속성

    HTML5 표준에 따라 다음 중 하나 일 수 있습니다. true (속성이 존재 함) 또는false (속성이 존재하지 않습니다).

    쿼리 매개 변수 값 (다른 경우에만 iframe)이 승리하는 다른 속성과 달리 속성은 Video.js에서 승리하기 때문에 여기에서 승리합니다.

    <video-js autoplay controls></video-js>
  2. URL 해시 또는 쿼리 문자열

    대부분의 경우와 달리 자동 재생 쿼리 문자열 매개 변수 또는 URL 해시 매개 변수는 이전 버전과의 호환성을 위해 모든 유형의 삽입에 적용됩니다.

    향후 주요 릴리스에서 인 페이지 삽입을 위해 제거해야합니다.

    매개 변수 존재는 다음과 같습니다. true , 즉?autoplay&foo=bar , 그러나 문자열 값이 지원됩니다. 예 : ?autoplay=muted&foo=bar

    자동 재생을 다음으로 설정할 수 없습니다. false쿼리 / 해시 매개 변수를 통해!

  3. videojs() / bc()옵션

    자동 재생 옵션은bc()함수.

  4. JSON 구성

    옵션에서 자동 재생을 사용할 수없는 경우 플레이어의 JSON 구성에서 전달 된 값을 사용합니다.

  5. 기본

    마지막으로, 위의 어느 것도 적용되지 않으면 기본값으로false .

브라우저 설정 및 정책

최근 몇 년 동안 브라우저의 일관성이 향상되었지만 다음 섹션에서는 각 주요 브라우저의 사용 가능한 설정 및 / 또는 정책을 요약합니다.

Safari (macOS)

macOS의 Safari에는 다음과 같은 설정이 있습니다.

  • 모든 자동 재생 허용
  • 소리로 미디어 중지 (기본값)
  • 자동 재생 안 함

또한 이러한 설정은 사이트별로 변경할 수 있습니다.

Safari (iOS 및 iPadOS)

iOS 및 iPadOS의 Safari에는 사용자 용 자동 재생 설정이 없지만 다음 정책을 구현합니다.

  • 음소거 된 자동 재생은 항상 작동합니다.
  • 소리와 함께 자동 재생하려면 사용자 제스처가 필요합니다.
  • iPhone에서는playsinline속성은 모든 경우에 필수입니다.

읽다이 WebKit 블로그 게시물자세한 내용은.

크롬

Chrome에는 사용자 용 자동 재생 설정이 없지만 다음 정책을 구현합니다.

  • 음소거된 자동 재생은 항상 작동해야 하지만언제나브라우저까지; 우리는 그렇지 않은 몇 가지 극단적인 경우를 만났지만 우리는 그것들을 통제할 수 없습니다.
  • 소리와 함께 자동 재생하려면 사용자 제스처 또는 충분히 높은 MEI(데스크톱에서)가 필요합니다.

읽다이 Chrome 블로그 게시물자세한 내용은.

파이어폭스

Firefox에는 다음과 같은 설정이 있습니다.

  • 오디오 및 비디오 허용
  • 오디오 차단 (기본값)
  • 오디오 및 비디오 차단

에지

기본적으로 Edge에는 다음 설정이 있습니다.

  • 허용 (기본값)
  • 제한

그러나edge://flags/#edge-autoplay-user-setting-block-option , Edge (Legacy)에서 사용할 수 있었던 자동 재생 차단 옵션을 복원 할 수 있습니다.

Edge (레거시)

Edge (Legacy)에는 다음과 같은 설정이 있습니다.

  • 허용
  • 제한
  • 블록

제한 옵션은 음소거 된 자동 재생 또는 사용자 제스처가 필요하다는 점에서 다른 브라우저와 유사합니다.

알려진 문제

인 페이지 소스 및 프로 그래 매틱 자동 재생

인 페이지 소스를 사용할 때 (예 : <source>요소) 및 호출play()또는 문자열 기반 자동 재생 값 사용- "play" , "muted" , 및"any" .

이것은 Firefox에서 보였지만 다른 브라우저에서도 나타날 수 있습니다.

이는 Brightcove 고객에게 일반적인 사용 사례가 아닙니다. 플레이어 생성 후 Video Cloud 소스가로드되기 때문에이 문제가 발생하지 않습니다.

대신, 당신은canplay호출하기 전에 실행할 이벤트play() . 또는 구성autoplay: true사용자 설정 / 기본 설정에 따라 예상대로 작동합니다.

IMA3 / FreeWheel

자동 재생 문자열 값

특수 문자열 기반 자동 재생 값- "play" , "muted" , 및"any" -있습니다현재 지원되지 않음 IMA3 및 FreeWheel 플러그인과 함께.

그러나 이러한 플러그인에는 자체 자동 재생 처리 기능이 있습니다. autoplay: true구성되면 마치autoplay: "any"구성되었습니다.

누락되거나 결함이있는 프리 롤

전화를 시도하는 경우play()이러한 플러그인을 사용하면 자동 재생이 실패하고 재생 버튼을 클릭해도 프리 롤이 재생되지 않는 경우가 있습니다.

FreeWheel을 사용하면 프리롤이 재생된다는 보고도 있습니다. 뒤에 큰 재생 버튼.

권장 사항

이러한 플러그인을 사용할 때 권장되는 자동 재생 활성화 방법은 다음과 같습니다.

  • 플러그인을 플레이어에 번들로 묶습니다 (예 : Studio를 통해). 사이드 로드(인페이지)는 권장되지 않습니다..
  • 특수 자동 재생 값을 사용하지 마십시오. 자동 재생 설정true광고 플러그인이 재생을 시도하도록 허용합니다.
