소개

기본적으로 Brightcove Player는 Brightcove Playback API와 통신합니다. 재생 권한 및 제한을 관리하는 새로운 시스템이 Playback API 앞에 있으며 DRM 라이선스를 사용하여 재생 권한을 제공합니다.

DRM을 사용하거나 사용하지 않고 재생 권한을 사용할 수 있지만 런타임 제한을 사용하도록 선택한 경우 JWT(JSON Web Token)를 사용해야 합니다.

License Keys Protection은 DRM으로 보호되거나 HLSe(HTTP Live Streaming Encryption) 콘텐츠와 함께 Dynamic Delivery를 사용할 때 추가 보안 수준을 제공합니다. 라이선스 요청은 서명된 JSON 웹 토큰 (JWT) 을 사용하여 인증할 수있습니다. 토큰은 비디오가 플레이어에 로드되고 소스가 선택되면 비디오 라이센스를 요청할 때 사용됩니다.

라이선스 키 보호를 위해 Brightcove Player를 구성하려면 비디오에 대한 카탈로그 요청을 할 때 토큰 매개 변수를 전달합니다.

이 기능에 익숙하지 않은 경우개요를 참조하십시오. 브라이트코브 재생 제한문서

다음 섹션의 코드를 이해하는 데 도움이 되도록 다음 문서의 개념을 검토하십시오.

  • JSON 웹 토큰 (JWT) 생성
  • 개념: 자바스크립트 프로미스 사용
  • 카탈로그 매개변수
  • DRM용 브라이트코브 플레이어 구성

요구 사항

다음은 재생 제한을 사용하는 데 필요한 요구 사항입니다.

재생 권한

재생 권한을 사용하려면 다음이 필요합니다.

  • 플레이어 버전 6.39.0이상
재생 권한을 사용할 때 정책 키로 구성된 콘텐츠 제한은 무시됩니다.

라이센스 키 보호

라이센스 키 보호를 사용하려면 다음이 필요합니다.

  • 플레이어 버전 6.33.0이상
  • DRM을 사용하는 경우 DRM 플러그인버전 5.5.0이상

재생 권한 사용

재생 권한을 활용하려면 다음 단계를 따르세요.

  1. Brightcove Player의 정책 키를 제거하십시오. 자세한 내용은재생 권한으로 요청하기부분.

  2. 당신이 가지고 있다면런타임 제한 , 인증 토큰이 필요합니다. 지정된 경우 이 토큰은 모든 후속 요청에 대한 권한 부여 헤더로 추가됩니다.

    다음과 같이 플레이어에 추가할 수 있습니다. 

    player.catalog.setBcovAuthToken('your jwt token');
  3. 정책 키 및/또는 인증 토큰을 변경한 후에는 Brightcove Playback API에서 데이터를 요청하고 플레이어에 로드할 준비가 된 것입니다. 이 과정은 기본 케이스와 동일합니다.

    다음은 재생 제한 및 인증 토큰이 있는 단일 동영상을 가져오는 예입니다.

    주의 bc() 메서드카탈로그를 호출하기 전에 정책 키를 설정할 수 있도록 하는 데 사용됩니다.

    HTML

    <div style="max-width: 960px;">
  <video-js id="myPlayerID"
    data-embed="default"
    controls=""
    data-application-id=""
    class="vjs-fluid"></video-js>
</div>
<script src="https://players.brightcove.net/your account id/your player id_default/index.min.js"></script>

    자바 스크립트

    <script>
    // +++ Add the player attributes +++
    var myPlayer,
    	myPlayerEl = document.getElementById("myPlayerID");
    myPlayerEl.setAttribute('data-account', your account id);
    myPlayerEl.setAttribute('data-player', 'your player id');

    // +++ Create the player +++
    myPlayer = bc(myPlayerEl);

    // Unset the player policy key
    myPlayer.catalog.setPolicyKey(null);

    // Set the authorization token
    myPlayer.catalog.setBcovAuthToken('your jwt token');
    // This should trigger a request to:
    //
    //   https://edge-auth.api.brightcove.com/playback/v1/videos/1
    //
    // With header:
    //
    //   Authorization: Bearer <span class="bcls-input">your jwt token</span>
    //

    myPlayer.catalog.get({id: 'your video id', type: 'video'}).
    then(function(data) {
      myPlayer.catalog.load(data);
      myPlayer.muted(true);
      myPlayer.play();
    }).
    catch(function(error) {
      throw new Error(error);
    });
</script>

재생 권한으로 요청하기

기본적으로 모든 Brightcove 플레이어는 정책 키가있는 경우 Playback API에 요청합니다. 재생 권한을 먼저 확인하는 요청을하려면 정책 키를 제거해야합니다. 이를 수행하는 두 가지 방법이 있습니다.

모든 플레이어에서 정책 키 제거

Video Cloud Studio에서 생성 된 새 플레이어에 대한 정책 키 수집을 방지하려면 다음을 수행하십시오.

  1. 계정 관리자에게 문의하십시오.
  2. 계정 ID를 제공하고remove policy key깃발.

개별 플레이어에서 정책 키 제거

기본적으로remove policy key귀하의 계정에 플래그가 비활성화되어 있습니다. 개별 플레이어에서 정책 키를 제거하려면 다음을 수행하십시오.

  1. 비디오 클라우드 스튜디오에서플레이어모듈로 이동합니다.
  2. 정책 키를 제거 할 플레이어를 선택하십시오.
  3. 왼쪽 탐색에서 JSON 편집기를 선택합니다 .
  4. 설정policy_key필드로null .
  5. 변경 사항을 저장하고 플레이어를 게시합니다.

재생 권한없이 요청하기

이 프로세스는 권한 및 제한없이 Playback API로 다시 요청을 보내도록 되돌릴 수 있습니다.

정책 키와 권한 부여 토큰을 다음과 같이 설정합니다.

  1. 만약 당신이remove policy key플래그를 활성화 한 다음 계정 관리자에게 문의하여 비활성화하십시오.

    당신이 가지고있는 경우remove policy key플래그를 활성화하고 비활성화하면 정책 키가 설정한 플레이어를 제외한 모든 플레이어에 대해 재생성됩니다. policy_key필드null .
  2. Studio에서 개별 플레이어를 업데이트 한 경우 정책 키로 JSON을 업데이트합니다.
  3. 인증 토큰을 다음으로 설정null . 
    player.catalog.setBcovAuthToken(null);

라이센스 키 보호 사용

License Keys Protection을 사용하려면 인증 토큰 문자열을플레이어 카탈로그객체, 속성 이름 사용bcovAuthToken .

이 접근 방식은 DRM 및 HLSe 콘텐츠 모두에 적용됩니다. 플레이어는 Playback API에서 로드되는 소스 유형을 감지하고 해당 소스에 대한 올바른 구현을 제공합니다.

라이센스 키 보호를 사용하려면 다음 단계를 따르십시오.

  1. 서명된 JSON 웹 토큰 (JWT) 을 만듭니다 .
  2. 이 토큰을 비디오 요청에 포함시킵니다.

    이 샘플 구현 코드는 catalog.get ()메서드를 사용하여 토큰을 제공하는 동안 비디오를 요청합니다.

    data-video-id매개변수는<video-js>태그에 포함시키지 않도록 해야 합니다.
      <video-js id="myPlayerID"
    data-account="1507807800001"
    data-player="default"
    data-embed="default"
    controls
    data-application-id></video-js>
  <script src="//players.brightcove.net/1507807800001/default_default/index.min.js"></script>
  
  <script>
    (function() {
      var myPlayer = videojs.getPlayer('myPlayerID');
  
      myPlayer.catalog.get({
        type: 'video',
        id: '6015247091001',

      })

          myPlayer.catalog.load(videoReturned);
        })
        .catch(function(err){

동시성 및 등록 사용

사용하는 경우스트림 동시성또는장치 등록 , 그러면 Brightcove 플레이어에서 EME표준 대신 플러그인 DRM플러그인.

DRM 플러그인은 EME 플러그인의 래퍼입니다. 자세한 내용은 DRM용 Brightcove 플레이어 구성문서.

다음 단계를 따르십시오.

  1. 추가 EME플러그인을 Brightcove 플레이어에 연결합니다. 포함하도록 플러그인이 업데이트되었습니다. persistentState .
  2. 비디오 클라우드 스튜디오에서플레이어모듈로 이동합니다. 해당 정보를 보려면 플레이어 링크를 선택하십시오.
  3. 왼쪽 탐색에서재생 .

  4. 확인 DRM 활성화선택되지 않았습니다.

  5. 왼쪽 탐색에서 JSON 편집기를 선택합니다 .

  6. JSON 코드에서plugins다음과 같이 EME 플러그인을 추가합니다.

    "plugins": [
  {
  "name": "eme",
  "scripts": [
    "//players.brightcove.net/videojs-drm/5.9.1/videojs-drm.min.js"
  ],
  "stylesheets": [
    "//players.brightcove.net/videojs-drm/5.9.1/videojs-drm.css"
  ],
  "options": {
    "keySystems": {
      "com.widevine.alpha": {
        "persistentState": "required"
      }
    }
  }
},
    EME 플러그인은 위에 표시된 순서대로 구성해야 합니다(스크립트 및 스타일 시트를 먼저 선택한 다음 옵션). 그렇지 않으면 작동하지 않습니다.
    이 기능은 DRM 플러그인 v5.9.1 이상에서 지원됩니다.
  7. 에 정의된 대로 JWT 인증 토큰을 포함합니다. 라이센스 키 보호 사용부분.

사용자 지정 구현 사용

당신은 당신이 가지고 있지 않은 사용자 정의 구현을 사용하고있을 수 있습니다bcovAuthToken값을 설정하려면카탈로그.get()메소드 요청. 본인 플레이어 또는 타사 플레이어를 사용하는 경우 다음 방법 중 하나를 사용하여 토큰을 라이선스 요청에 전달할 수 있습니다.

  • HTTP 헤더: BCOV-Auth (HLSE에서는 지원되지 않음)
  • 쿠키: bcov-auth (HLSE에서는 지원되지 않음)
  • 쿼리 매개 변수: bcov-auth (HLSE에서만 지원) 라이센스 URL 대신 마스터 매니페스트 URL에 추가해야 합니다.

다음은 video 객체의source.emeHeaders['BCOV-Auth']속성을 토큰으로 설정하는 방법을 보여주는 예제입니다. 이렇게 하면emeHeader카탈로그 요청 후 각 소스에 가 삽입됩니다. 

<video-js id="myPlayerID"
  data-account="1507807800001"
  data-player="default"
  data-embed="default"
  controls
  data-application-id></video-js>
<script src="//players.brightcove.net/1507807800001/default_default/index.min.js"></script>

<script>
  (function() {
    var myPlayer = videojs.getPlayer('myPlayerID');

    myPlayer.catalog.get({
      type: 'video',
      id: '6015247091001'
    })
    .then(function(video){
      sources=video.sources;

      for (let i = 0; i < sources.length; i++) {
        const source = sources[i];

        // Only add the auth token as an eme header for DRM content
        if (your jwt token && source.key_systems) {
          source.emeHeaders = {
              'BCOV-Auth': your jwt token
            };
        }
      }
        myPlayer.catalog.load(video);
      })
      .catch(function(err){
        console.log('err:', err);
      });
  })();
</script>
