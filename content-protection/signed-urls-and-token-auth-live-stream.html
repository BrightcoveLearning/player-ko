라이브 스트림으로 서명 된 URL 및 토큰 인증

{{ page.description }}

개요

프리미엄 또는 기밀 콘텐츠를 제공하는 경우 보안을 유지할 수 있는 모든 방법을 모색하게 됩니다. 최종 사용자가 적절한 권한 부여 없이는 비디오 URL을 복사하거나 콘텐츠를 계속 재생할 수 없도록 합니다. Brightcove는 RTMP, HLS 및 DASH URL 서명을 지원합니다.

자체 토큰을 생성하려는 경우 이 프로세스에 한 단계가 있습니다. 라이브 시스템이 토큰화된 URL을 생성하도록 하려면 라이브 스트림에 대한 토큰 인증을 활성화하는 데 필요한 두 단계가 다음 섹션에 자세히 설명되어 있습니다.

현재 토큰 인증은 Akamai EdgeAuth 2.0에서만 지원됩니다.

구현

참고: 고유한 토큰을 만들어 URL에 추가하는 경우 1단계만 완료해야 합니다.

1 단계: 라이브 작업 만들기

토큰 인증을 활성화하려면 요청 본문에 다음을 포함하십시오. 일자리를 만들다 :

  "add_cdns": [{
    "label": "My CDN",
    "prepend": "mycdn-a.akamaihd.net",
    "protocol": "https",
    "vendor": "akamai",
    "token_auth": {
      "auth_type": "Akamai2.0",
      "key": "AKAMAI_TOKEN_PASSWORD",
      "token_name": "AKAMAI_TOKEN_NAME",
      "media": {
        "start_time": "now",
        "end_time": 1495697388,
        "ttl": 60
      }
    }
  }]

관련 필드는 다음 표에 설명되어 있습니다.

라이브 작업을 위한 TTL 필드
필드 가치
start_time 어느 하나now또는 유닉스/포식스 시작 시간
end_time 유닉스/포식스 종료 시간
ttl TTL(Time to Live)의 초 단위 숫자

2 단계: 라이브 작업이 생성된 후

다음과 같은 경우에만 이 단계를 완료하십시오. ~ 아니다자신의 토큰을 만들고 대신 Live API가 토큰화된 URL을 생성하기를 원합니다.

라이브 작업이 생성된 후 마스터 매니페스트 URL을 생성하기 위해 다른 요청을 해야 합니다. 이렇게 하려면POST요청:

  https://api.bcovlive.io/v1/jobs/JOB_ID/authurls

다음 JSON을 요청 본문으로 포함합니다.

  {
    "ttl": number in seconds for time to live
  }

노트

  • NS ttl위의 두 단계에 대한 값은 DVR 창보다 크거나 같아야 합니다. 그렇지 않으면 DVR 창은 다음으로 제한됩니다. ttl값
  • http와 https가 모두 지원됩니다.
  • NS token_name 5~12자여야 하며 Akamai에 구성된 것과 일치해야 합니다.
  • start_time선택 사항이며 될 수 있습니다"now" (기본값) 또는 초 단위의 Epoch 시간 값
  • end_time선택 사항이며 초 단위의 에포크 시간 값일 수 있습니다. end_time또는ttl필수입니다. 지정되지 않은 경우ttl더 큰 것으로 가정된다. live_dvr_sliding_window_duration그리고vod_sliding_window_duration
