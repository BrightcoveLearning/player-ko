개요: 브라이트코브 플레이어

{{ page.description }}

강점

플레이어는 유연하고 확장 가능한 비디오 재생 환경을 구축 할 수있는 플랫폼을 제공합니다.

주요 강점은 다음과 같습니다.

  • 가벼운 HTML5 동영상 플레이어
  • 기능을 추가 할 수있는 풍부한 플러그인 환경
  • HTML, JavaScript 및 CSS를 사용하여 플레이어 모양 사용자 지정
  • 서로 다른 환경을위한 두 가지 플레이어 구현
  • 큐 포인트, 캡션, 현지화, 광고 등과 같은 풍부한 기능 세트
  • 네이티브 플레이어 SDK
  • 인터넷 액세스 URL로 Video Cloud 라이브러리 비디오 및 비디오를 모두 재생합니다.
이 문서를 보고 있는 이 사이트, https://player.support.brightcove.com , 를 위한 것입니다편물브라이트코브 플레이어 버전. Brightcove Player의 SDK 버전에 대한 정보는 다음에서 찾을 수 있습니다. https://sdks.support.brightcove.com/ . 또한 스타일 지정 또는 특수 구성과 같은 웹 버전의 변경 사항은 SDK 버전에 자동으로 적용되지 않으며 그 반대의 경우도 마찬가지입니다.

동영상 게시

플레이어를 만들 때 플레이어 게시를위한 세 가지 구현이 표시됩니다. 이러한 구현은 다음과 같습니다.

  • URL
  • 인페이지 임베드
  • 이프레임

이러한 각 구현은 두 가지 접근 방식을 취할 수 있습니다. 가장 일반적인 접근 방식은 동적 플레이어를 사용하는 것입니다. 이 플레이어는 다양한 비디오를 재생할 수 있습니다. 이를 위해 동영상 ID는 URL 매개 변수, 동영상 태그 속성 또는 자바 스크립트 API를 통해 런타임에 플레이어에 전달됩니다. 이는 다양한 콘텐츠 인스턴스에 사용할 수있는 템플릿 웹 페이지와 기술적으로 유사합니다. 이 접근 방식의 사용 사례는 사용자가 액세스하는 아티클 또는 비디오에 따라 다른 비디오가 할당되는 일반 비디오 플레이어입니다.

보다 전문적인 접근 방식은 동영상을 플레이어 구성 자체에 포함하여 플레이어에 정적으로 바인딩하는 것입니다. 이 접근 방식은 정적 웹 페이지를 만드는 것과 유사합니다. 장점은 포스터 이미지를 표시하거나 비디오 재생을 시작하기 위해 플레이어가 수행해야하는 호출 수를 줄여 성능을 향상시키는 것입니다. 이 작업은 Studio에서 수행하거나 API를 직접 사용할 수 있습니다. 이 접근 방식의 사용 사례는 단일 제품을 자세히 설명하는 비디오가 포함 된 플레이어로 구축 된 제품 페이지입니다.

참고: 플레이어에 정적으로 바인딩 된 비디오를 변경할 수는 있지만 좋은 방법은 아닙니다. 이렇게 하면, 동적으로 추가된 비디오가 표시되기 전에 정적으로 바인딩된 비디오의 포스터가 플레이어에 잠시 표시되는 것을 볼 수 있습니다. . 이러한 이유로 동영상이 동적으로 변경되는 상황에서 사용하기 위해 처음에 연결된 동영상이 없는 플레이어를 만드는 것이 가장 좋습니다.

플레이어 관리

 

플레이어는 Studio를 통해 또는 플레이어 관리 API를 사용하여 프로그래밍 방식으로 생성, 사용자 지정 및 관리 할 수 있습니다.

스튜디오

Studio는 이러한 작업을 수행하기 위해 사용하기 쉬운 인터페이스를 제공합니다.

그만큼 Studio 시작하기문서는 Video Cloud Studio에 대해 배울 수있는 훌륭한 출발점을 제공합니다.

플레이어 관리 API

플레이어 관리 API의 기능은 크게 두 가지 범주로 나뉩니다. 이러한 범주와 세부적인 강점은 다음과 같습니다.

  • 중앙 집중식 관리
    • 자동 버전 업데이트
    • API를 통해 플레이어 생성, 업데이트 및 활성화
    • 한 번에 수천 명의 플레이어 업데이트
    • 플레이어 버전 관리 및 롤백
  • 최적화 된 성능
    • 플레이어 자산의 사전 컴파일
    • 플레이어에 썸네일 또는 비디오 URL 삽입
    • 동일한 CDN을 통해 플러그인 제공

플레이어 관리 API에 대한 자세한 정보는 Brightcove 플레이어 관리선적 서류 비치.

정적으로 바인딩 된 플레이어 게시

위에서 언급했듯이 비디오를 플레이어에 정적으로 바인딩 할 수 있습니다. 이는 플레이어 UI 또는 플레이어 관리 API를 통해 수행 할 수 있습니다. 자세한 내용은비디오 및 재생 목록 게시 미디어 모듈 사용그리고단계별 : 플레이어 관리서류.

이 경우 동영상 ID는 iframe 또는 직접 링크에 사용되는 동영상 URL에 포함되지 않아도되지만<video>인 페이지 삽입을위한 태그입니다.

플레이어 유형 비교

Brightcove의 플레이어 환경을 구성하는 플레이어 유형에는 세 가지가 있습니다. 그들은 다음과 같습니다.

  • Video.js : HTML5 비디오를보다 쉽게 작업하고 빌드 할 수 있도록 프레임 워크를 제공하는 오픈 소스 JavaScript 및 CSS 라이브러리입니다. Video.js 프로젝트는 Brightcove가 후원하며 Brightcove는 프로젝트에 크게 기여합니다.
  • Brightcove Player (Studio의 플레이어 모듈) : Video.js를 기본 플레이어 프레임 워크로 사용하는 Brightcove에서 개발 한 비디오 플레이어입니다. Video.js를 단독으로 사용할 수 있지만 대부분의 경우 추가 플러그인 및 사용자 정의와 결합하고 싶을 것입니다. Brightcove는 플러그인을 개발하고 호환성을 관리하며 시간이 지남에 따라 모든 것을 최신 상태로 유지합니다. Brightcove Player는 모든 Brightcove 비디오 경험의 기반을 형성합니다.
  • Brightcove 플레이어 : 비디오 플레이어 경험을 만들고 관리하기위한 고성능 서비스입니다. 세계 최고의 비디오 게시자의 요구를 충족하도록 설계된 Brightcove는 선도적 인 크로스 플랫폼 HTML5 우선 비디오 플레이어 인 Brightcove Player와 강력한 플레이어 관리 API 및 성능 최적화 서비스 세트를 결합합니다. Brightcove의 기본 iOS 및 Android SDK도 포함되어 있습니다.

다음 정보는 오픈 소스 Video.js 대신 Brightcove Player를 선택하는 것이 유리할 수있는 이유를 설명합니다.

  • 성능 최적화 : 플레이어 관리 서비스는 플레이어 제공을 최적화하고, 플러그인을 미리 컴파일하고, 에셋을 스키닝하고, 썸네일을 스키닝하여 각 플레이어를 최적화하여 다운로드 크기를 최소화 한 다음, 고성능의 글로벌 최적화 CDN을 통해 제공합니다.
  • 플레이어 관리 : 플레이어 관리 기능을 통해 사용자는 플레이어 구성을 쉽게 추적하고 변경할 수 있습니다. 이는 사이트 또는 애플리케이션의 다른 부분에 대해 다양한 사용자 지정 및 플러그인 및 플레이어 스타일을 포함하여 여러 플레이어 구성을 사용하는 고객에게 특히 유용합니다. 플레이어 관리 서비스를 사용하면 각 페이지의 코드를 변경하지 않고도 플러그인, 스타일 및 기타 설정을 변경하는 플레이어의 구성을 업데이트할 수 있습니다. 고객은 간단히 플레이어를 다시 게시 할 수 있으며 이러한 변경 사항은 자동으로 CDN으로 푸시됩니다.
  • Future Proof 타사 통합 : Brightcove는 comScore, Nielsen, Google / Doubleclick, Freewheel 등과 같은 파트너와 직접 협력하여 분석 및 광고 플러그인이 제대로 작동하는지 확인합니다. 또한 Brightcove는 브라우저와 연결된 장치의 모든 트렌드를 최신 상태로 유지하여 플레이어가 타사 웹 기술에 대한 모든 업데이트를 완벽하게 작동 할 수 있도록합니다. Brightcove 플러그인에서 제공하는 일부 기능은 DFP 광고를 표시하는 데 사용되는 Google의 IMA SDK를 사용하여 광고 표시를 위해 Flash와 HTML5 간 자동 전환과 같이 오픈 소스에서 사용할 수 없습니다. Brightcove는 플러그인이 모든 버전의 Video.js가 아닌 Brightcove Player로 테스트되도록합니다.
  • 업계 최고의 지원 : Brightcove Player를 선택하는 또 다른 중요한 이유는 서비스가 Brightcove에서 지원되기 때문입니다. 일반적으로 이것은 고객으로서 모든 단계에서 도움을 줄 경험있는 파트너가 있음을 의미합니다. Brightcove를 사용하면 시장 출시 시간을 단축하고 작동이 보장되는 시스템을 확보 할 수 있으며 그렇지 않거나 질문이있을 때 전화 할 사람이 있습니다.
  • 개발 비용 및 출시 시간 : Brightcove가 수행 한 것처럼 HTML 비디오 플레이어에 대한 모든 기능을 직접 구축 할 수는 있지만 저렴하지 않으며 시간이 많이 걸립니다. 이런 종류의 작업은 Brightcove가 판매하는 회사의 핵심 역량이 아닐 가능성이 높으므로 회사가 Brightcove만큼 또는 더 적은 비용으로이를 수행 할 수있을 가능성은 낮습니다. 오픈 소스는 처음에는 저렴 해 보이지만 완전한 솔루션을 구축하는 데 훨씬 더 많은 시간과 노력이 필요하며 플레이어, 플러그인 및 통합을 최신 상태로 유지하기 위해 지속적인 유지 관리 비용이 발생합니다.
  • 통합 후 : Brightcove Player는 스트림으로 인코딩 된 서버 측 광고를 사용하여 Once 스트림의 재생을 지원하도록 특별히 제작되었습니다. 또한 플레이어는 Once UX와 통합되어 VMAP 콘텐츠를 기반으로 한 풍부한 대화 형 광고 경험을 가능하게하여 광고가 서버 측에서 연결되는시기를 알려줍니다.
