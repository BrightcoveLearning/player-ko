단계별: 브라이트코브 플레이어

{{ page.description }}

개요

이 문서는 Brightcove Player를 만든 다음 두 가지 다른 코드 구현을 사용하여 새로 만든 플레이어를 사용하여 비디오를 게시하는 방법에 대한 실습 소개를 제공합니다. 또한 프로그래밍 방식으로 비디오 재생을 시작한 다음 플레이어에 플러그인을 추가합니다.

이 빠른 시작에서는 다음 작업을 수행합니다.

  • Studio를 사용하여 플레이어 만들기
  • Studio를 사용하여 플레이어에 비디오 게시
  • iframe 플레이어 구현 사용
  • 인 페이지 삽입 구현 사용
  • 프로그래밍 방식으로 비디오 재생
  • 플레이어에 오버레이 플러그인 추가

플레이어 만들기

새 플레이어를 만들려면 다음 단계를 따르십시오.

  1. 다음에서 Studio에 로그인하십시오. https://studio.brightcove.com/ .
  2. 현재 계정 이름이 페이지 오른쪽 상단에 표시됩니다. 계정이 여러 개인 경우당좌 계정링크를 클릭하고 플레이어를 만들려는 계정을 선택하십시오.
  3. 탐색 헤더에서플레이어링크를 클릭합니다.
  4. [ 새 플레이어]버튼을 클릭합니다.
  5. 플레이어의이름과간단한 설명을입력합니다. 이 빠른 시작을 위해 플레이어 이름은샘플 플레이어 .
  6. 저장을 클릭합니다 .
  7. 새로 생성 된 플레이어가 플레이어 목록에 나타나는지 확인합니다.

플레이어에 비디오 게시

미디어 모듈에서 새로 만든 플레이어에 비디오를 게시합니다. 비디오를 플레이어와 연결하면 비디오를 게시하고 플레이어에서 비디오를 볼 수 있는 세 가지 구현에 액세스할 수 있습니다.

플레이어에서 비디오를 게시하려면 다음 단계를 따르십시오.

  1. Studio에서 클릭하여 미디어 모듈로 이동합니다. 미디어메인 메뉴의 링크.
  2. 플레이어에 게시하려는 오디오 트랙이 없는 비디오 행을 클릭한 다음게시 및 포함> 웹에 게시 .
    참고: 비디오에 오디오 트랙이 있는 경우 자동으로 재생되지 않습니다. 이 트랙은 나중에 문서에서 발생하기를 원합니다. 문서 참조자동 재생 고려 사항사용자 상호 작용 없이 재생되는 동영상에 대한 자세한 내용은
  3. 로부터선수 선택드롭 다운 메뉴에서 새로 만든 플레이어를 선택합니다.
  4. 이제 플레이어의 세 가지 구현에 액세스 할 수 있습니다. 표준 (iframe), 많은 (인 페이지) 및브라우저에서 미리보기 URL. 이 빠른 시작에서 나중에 여러 번 코드를 복사하므로이 창을 닫지 마십시오.
참고: 궁금한 경우 URL 코드를 사용하여 플레이어 구성의 “아래”를 볼 수 있습니다. 클릭브라우저에서 미리보기 URL 링크. 브라우저에서 index.html URL 끝에 config.json . 방금 생성한 플레이어의 현재 구성을 자세히 설명하는 JSON 개체가 표시됩니다.

JSON 편집기를 사용하여 플레이어에 비디오를 추가합니다. 비디오 콘텐츠를 구성하려면 다음 단계를 따르십시오.

  1. 샘플 플레이어 링크를클릭하여 플레이어 속성 페이지를 엽니다.
  2. 왼쪽 탐색 메뉴에서 JSON 편집기를클릭합니다.
  3. JSON에서 SECOND TO THE LAST 중괄호 뒤에 쉼표를 넣으십시오.
  4. 마지막 닫는 중괄호 바로 위에 다음을 추가하여 플레이어에 비디오와 포스터를 할당합니다. 물론 자신의 비디오와 포스터를 대체 할 수 있습니다. 
     "media": { "sources": [{
 "src": "//solutions.brightcove.com/bcls/assets/videos/BirdsOfAFeather.mp4", "type": "video/mp4"
 }],
 "poster": {
 "highres": "//solutions.brightcove.com/bcls/assets/images/BirdsOfAFeather.jpg"
 }
 }
  5. JSON은 다음과 유사하게 나타나야합니다 (추가 된 텍스트는 노란색으로 강조 표시됨).
  6. 저장을 클릭합니다 .
플레이어를 게시하면 게시된 플레이어에 변경 사항이 푸시됩니다. 미리보기 플레이어, 게시 된 플레이어 및 플레이어 게시 코드에 대한 자세한 내용은플레이어 삽입 생성 암호 . 플레이어를 게시하려면 다음 단계를 따르세요.
  1. 딸깍 하는 소리게시 및 삽입그리고변경 사항 게시플레이어를 게시합니다.
  2. 변경 사항을 게시 한 동일한 대화 상자에서 표준 (iframe) 또는 고급 (인 페이지 임베드) 플레이어 구현 코드를 복사 할 수도 있습니다. 구현 코드는사이징 ( 반응형 또는 수정됨 ), 종횡비 과 치수 선택 및 저장선수 정보옵션.
  3. 나중에 사용할 수 있도록 두 코드 구현 유형을 모두 텍스트 파일에 복사합니다.

표준 코드 사용

참고: 표준 코드 (iframe 구현)의 경우 iframe 요구 사항으로 인해 웹 서버를 사용하여 페이지를 탐색해야합니다. 로컬에서 페이지를 보는 것만으로는 작동하지 않습니다.
  1. HTML 페이지를 만들고표준 (iframe) 코드를 이전 단계에서 본문으로 가져옵니다.
  2. 이 페이지를 탐색하면 비디오에크기 조정이 반영됩니다 ( 반응형 또는 수정됨 ), 종횡비및치수선택

고급 삽입 사용 암호

이 섹션에서는 고급 (인 페이지 임베드) 코드 구현을 사용합니다.

  1. Video Cloud Studio의 게시 대화 상자로 돌아갑니다.
  2. 보기고급의코드를 클릭하고결정된라디오 버튼사이징 .
    NS 반응형크기 조정에는 플레이어를 반응형으로 만들기 위한 HTML이 포함됩니다. 이 활동에서는 플레이어 코드에 초점을 맞추므로 더 간단한결정된사이징.
  3. HTML 페이지를 만들고많은 (인 페이지 삽입) 코드를 위에서 본문에 삽입합니다.
  4. 다음과 유사하게 표시되는 코드를 검토하고 속성을 확인합니다. 
     <video-js data-account="1507807800001"
  data-player="EUYJo0AOB"
  data-embed="default"
  controls=""
  data-video-id="6071787405001"
  data-playlist-id=""
  data-application-id=""
  width="640" height="360"></video-js>
<script src="https://players.brightcove.net/1507807800001/default_default/index.min.js"></script>
    • data-account : 계정 번호
    • data-player : 브라이트코브 플레이어의 ID
    • data-embed : 플레이어가 부모 임베드 또는 부모 플레이어의 ID인default경우, 임베드에대한 자세한 내용은 임베드 API 가이드 (부모/자식 플레이어) 를 참조하십시오.
    • controls : 플레이어 컨트롤이 존재해야 하거나 (태그에 포함된controls속성) 표시되지 않는 경우 부울 역할을 합니다 ( controls어트리뷰트는태그)
    • data-video-id : Video Cloud 동영상의 ID(플레이어 ID를 제공하는 경우 사용하지 마십시오)
    • data-playlist-id : Video Cloud 재생 목록의 ID(비디오 ID를 제공하는 경우 사용하지 않음)
    • data-application-id : 단일 플레이어를 재사용 할 수 있지만 사이트 별 또는 애플리케이션별로 분석을 차별화 할 수 있습니다. 보다플레이어 삽입에 애플리케이션 ID 추가 암호자세한 내용은
    • width & height : 플레이어 크기를 설정합니다.
    • <script>꼬리표: 인 페이지 코드를 사용할 때마다script꼬리표; 참조 된 JavaScript 파일에는 특정 플레이어와 관련된 모든 JavaScript 및 CSS가 포함되어 있습니다.
  5. 다음과 유사하게 표시되는 코드를 검토하고 속성을 확인합니다. 
     <video data-account="1507807800001"
 data-player="H15p1gTkg"
 data-embed="default"
 data-application-id
 class="video-js"
 controls></video>
 <script src="https://players.brightcove.net/1507807800001/H15p1gTkg_default/index.min.js"></script>
    • data-account : 계정 번호
    • data-player : 브라이트코브 플레이어의 ID
    • data-embed : 플레이어가 부모 임베드 또는 부모 플레이어의 ID인default경우, 임베드에대한 자세한 내용은 임베드 API 가이드 (부모/자식 플레이어) 를 참조하십시오.
    • class : 인 페이지 소스 코드와 관련된 표준 Brightcove Player CSS 클래스
    • controls : 플레이어 컨트롤이 존재해야 하거나 (태그에 포함된controls속성) 표시되지 않는 경우 부울 역할을 합니다 ( controls어트리뷰트는태그)
    • <script> tag : 특정 플레이어와 관련된 모든 관련 JavaScript 및 CSS
  6. 이 페이지를 탐색하면 비디오에크기 조정이 반영됩니다 ( 수정됨 ), 종횡비및치수선택

프로그래밍 방식으로 비디오 재생

Brightcove Player와 함께 사용할 수있는 풍부한 API가 있습니다. 이 섹션에서는play()프로그래밍 방식으로 동영상 재생을 시작하는 방법입니다.

  1. 추가id속성video값이있는 태그 myPlayerID . 
     <video id="myPlayerID"
  2. 마감 바로 위에body태그, 삽입script블록. 
     <script> </script>
 </body>
  3. 스크립트 블록에서on()청취하는 방법loadedmetadata행사. 이벤트가 전달되면 익명의 이벤트 핸들러 함수가 호출됩니다. 
     videojs.getPlayer('myPlayerID').on('loadedmetadata', function() { });
    참고: Brightcove Player의 시작 이벤트에 대한 자세한 내용은시작 이벤트섹션플레이어 이벤트 이해문서.
  4. 함수에 로컬 인 변수를 만듭니다. myPlayer , 플레이어 인스턴스를 할당하며 다음과 같이 참조됩니다. this , 해당 변수에. 
     var myPlayer = this;
  5. 대부분의 브라우저는 더 이상 오디오 트랙이있는 비디오의 자동 재생을 허용하지 않으므로 다음을 사용하는 것이 가장 좋습니다. muted()비디오를 음소거하고 비디오가 자동 재생되는지 확인하는 방법입니다. 
     myPlayer.muted(true);

    참조자동 재생 고려 사항자동 재생 문제에 대한 자세한 내용은 문서를 참조하십시오.

  6. 플레이어의play()비디오를 시작하는 방법. 
     myPlayer.play();
  7. 다음과 같이 스크립트 블록이 나타나는지 확인합니다. 
     <script> 
   videojs.getPlayer('myPlayerID').on('loadedmetadata', function() { 
     var myPlayer = this;
     myPlayer.muted(true);
     myPlayer.play();
   });
 </script>
  8. 변경 사항을 저장하고 브라우저에서 페이지를 새로 고치면 비디오가 자동으로 재생되기 시작합니다. 오디오 트랙이 포함 된 비디오를 선택한 경우 비디오를 재생하려면 재생 버튼을 클릭해야합니다.

플러그인 추가

Brightcove Player에는 플레이어를 향상시키기 위해 활용할 수있는 많은 플러그인이 있습니다. 이러한 플러그인 중 하나가 오버레이 플러그인입니다. 이 섹션에서는 마지막 섹션에서 완료한 고급 코드에 해당 플러그인을 추가합니다. 플러그인에 대한 자세한 내용은디스플레이 오버레이 플러그인문서.

플러그인을 사용할 때 플러그인 구현 인 JavaScript 경로를 제공해야합니다. 플러그인에서 필요한 경우 CSS에 대한 링크를 제공해야 할 수도 있습니다.

  1. 마감 바로 위에head태그에 오버레이 플러그인의 CSS 위치를 가리키는 다음 링크 문을 삽입합니다. 
     <link href="https://players.brightcove.net/videojs-overlay/lib/videojs-overlay.css" rel='stylesheet'> </head>
  2. 앞서 추가 한 스크립트 블록 바로 위에 다음을 추가합니다. script오버레이 플러그인 기능을 구현하는 실제 자바 스크립트 코드 인 태그입니다. 
     <script src="https://players.brightcove.net/videojs-overlay/lib/videojs-overlay.js"></script> <script>
 videojs.getPlayer('myPlayerID').on('loadedmetadata', function() {
  3. 바로 뒤에play()메서드가 사용되면 플레이어의overlay()메서드를 사용하고 매개 변수로 중괄호를 추가하여 JSON 구성 개체 전달을 준비합니다. 
     myPlayer.overlay({ });
  4. 여러 오버레이 개체를 배열로 플러그인에 전달하여 다양한 시간과 다양한 위치에 다른 콘텐츠를 표시 할 수 있습니다. 이 경우 일부 텍스트를 표시하기 위해 하나의 오버레이 개체 만 전달되며 동영상이 재생 될 때만 표시됩니다. 비디오가 일시 중지되면 오버레이가 표시되지 않습니다. 이를 구현하려면 다음 오버레이 개체를 구성 매개 변수로 추가합니다. 
     overlays: [{ 
  content: 'This event-triggered overlay message appears when the video is playing', 
  start: 'play',
  end: 'pause'
}]
  5. 스크립트 블록이 다음과 같이 나타나고 오버레이 플러그인이 올바르게 호출되고 구성되었는지 확인합니다. 
     <script> 
  videojs.getPlayer('myPlayerID').on('loadedmetadata', function() {
    var myPlayer = this;
    myPlayer.muted(true);
    myPlayer.play();
    myPlayer.overlay({
      overlays: [{
        content: 'This event-triggered overlay message appears when the video is playing', 
        start: 'play',
        end: 'pause'
      }]
    });
  });
</script>
  6. 변경 사항을 저장하고 브라우저에서 페이지를 새로 고치면 비디오가 자동으로 재생되기 시작합니다. 비디오가 시작되면 오버레이가 나타납니다. 오버레이가 제거되었는지 확인하려면 비디오를 일시 중지하십시오.
