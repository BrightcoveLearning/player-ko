단계별: 플레이어 사용자 지정

{{ page.description }}

개요

이 문서에는 인페이지 임베드 (in-page embed) 와 함께 CSS를 사용하는 것과 플레이어의 iframe 구현이 상당히 다르므로 여러 섹션이 있습니다.

이 문서의 콘텐츠는 Chrome의 개발 도구를 사용하지만 모든 최신 브라우저에는 동일한 기능이 있습니다. 여러 브라우저에서 개발 도구를 사용하는 방법에 대한 간략한 소개는디버깅 기본문서를 참조하십시오.

참고: 일부 기본 플레이어 스타일은 비디오 클라우드 스튜디오 내에서 제공됩니다. 자세한 내용은 플레이어스타일 지정을 참조하십시오 .

플레이어 선택기 결정

CSS 선택기를 사용하면 페이지에서 HTML 요소를 선택하고 조작할 수 있습니다. CSS 선택기는 ID, 클래스, 유형, 속성 등을 기반으로 HTML 요소를 “찾기" (또는 선택) 하는 데 사용됩니다. HTML 페이지에서 요소의 모양을 변경하려고 할 때 가장 먼저 해야 할 일은 요소를 처리하는 데 사용할 선택기를 결정하는 것입니다. 다음 단계에서는 플레이어의 선택기를 찾은 다음 플레이어의 크기를 변경하는 방법을 보여줍니다.

  1. Video Cloud Studio에서미디어모듈을 클릭하고 게시할 비디오를 선택합니다.
  2. 비디오를 재생할 플레이어를 선택하십시오.
  3. 설정사이징 , 종횡비플레이어 크기.
  4. 복사많은게시 코드.
  5. HTML 편집기를 사용하여 새 HTML 페이지를 만듭니다.
  6. 게시 코드를 HTML 페이지 본문에 붙여넣습니다.
  7. 새로 만든 페이지를 찾아봅니다.
  8. 브라우저의 개발 도구를 열고 HTML 코드를 검사하고 HTML로 드릴다운하여<video-js>꼬리표. 당신은 볼 것입니다<video-js>태그에는 이름이 지정된 클래스가 있습니다. video-js . 그만큼<video-js>태그는 항상video-js . 당신은video-js플레이어를 스타일링 할 때 자주 수업.
  9. 빨간색으로 강조 표시된 클래스를 보려면 아래 스크린샷을 확인하십시오. 녹색으로 강조 표시된 것은<video-js>물론 Brightcove Player입니다.

재생 버튼 선택기

또 다른 방법은 재생 버튼의 선택기를 찾는 데 사용됩니다. 다음 단계에서는 재생 버튼의 올바른 선택기를 찾은 다음 모양을 변경하는 방법을 보여줍니다.

  1. HTML 페이지를 탐색합니다.
  2. 플레이어를 오버레이하는 재생 버튼을 마우스 오른쪽 버튼으로 클릭하고(첫 번째 마우스 오른쪽 버튼을 클릭하면 Brightcove 플레이어 정보가 표시되므로 두 번 마우스 오른쪽 버튼을 클릭해야 함) 검사 .
  3. 참고<div>강조 표시된 태그는 재생 버튼을 나타내며vjs-big-play-button .
  4. 재생 버튼의 색상과 불투명도를 변경하는 CSS를 포함하도록 스타일을 변경합니다. 
    <style>
  .vjs-big-play-button {
    background-color: red;
    opacity: .3;
  }
</style>
  5. 페이지를 탐색하면 일부 스타일이 사용된 것은 아닙니다. 버튼이 빨간색이 아닙니다.
  6. 개발자 도구의 오른쪽을 보면 스타일 요약을 볼 수 있습니다. 아래를 살펴보면 배경을 빨간색으로 변경하는 것이 재정의되어 교차되는 것으로 표시됩니다.
    참고: Safari에서는 스타일이 오버라이드되더라도 스타일이 교차되지 않을 수 있습니다.

    무슨 일이 있었는지 배경색이 많이 변경되었습니다. 설정한 스타일은 CSS의 “계단식” 부분에서 재정의되었습니다. 특수성이라는 CSS의 기능을 활용하여 이를 변경할 수있습니다. 특수성은 브라우저에 의해 적용되는 CSS 규칙을 결정하며 CSS (계단식 스타일 시트) 규칙이 표시되는 방식을 나타내는 가중치 유형입니다. 특수성을 사용하면 스타일을 다른 스타일보다 더 중요하게 만들 수 있습니다. CSS 특이성에 대한 간략한 소개가 있습니다. 플레이어 모양 사용자 지정 doc 및 모든 검색에서 수많은 참조.

  7. 이 경우video-js클래스는 스타일이 우선하도록 "강화"합니다. 새 조합 선택기를 사용하도록 스타일을 변경합니다. 마지막 수업 전에 공백이 하나 있습니다 (이유에 대한 자세한 내용은 특정 규칙에서 찾을 수 있음). 
    <style>
  .video-js .vjs-big-play-button {
    background-color: #ff0000;
    opacity: .3;
  }
</style>
  8. 페이지를 탐색하고 이제 모든 스타일이 적용되었는지 확인합니다.
  9. 개발자 도구의 오른쪽을 다시 살펴보면 스타일 요약을 볼 수 있습니다. 이제 배경색 변경이 재정의되지 않은 것을 알 수 있습니다.

고급 코드 구현

위의 선택기 및 스타일로 작업 할 때 결과는 다음과 같이 플레이어의 고급 코드 구현과 함께 사용되었습니다.

<!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js .vjs-big-play-button {
      background-color: red;
      opacity: .3;
    }
  </style>
</head>

<body>

  <video-js data-account="1507807800001"
    data-player="default"
    data-embed="default"
    controls=""
    data-video-id="6065136830001"
    data-playlist-id=""
    data-application-id=""
    width="640" height="360">

  </video-js><script src="https://players.brightcove.net/1507807800001/default_default/index.min.js"></script>

</body>

</html>

이것은 HTML 페이지에서 CSS를 사용하는 매우 일반적인 방법입니다. 당신은<style>블록은 페이지에<video-js>꼬리표.

참고: CSS 경험에 따라 위의 코드가id속성<video-js>꼬리표. 그런 다음 다음과 같은 선택기를 사용할 수#myPlayer .vjs-big-play-button있습니다. 사실, 당신은 할 수 있고 효과가 있었을 것입니다. 문제는 iframe 플레이어 구현을 사용할 때 id 선택기를 사용하여 만든 CSS가 작동하지 않으므로 기존 클래스 이름만 사용된다는 것입니다. 사용하지 않음으로써id속성을 사용하면 스타일이 iframe 플레이어 구현에 직접 적용됩니다.

표준 코드 구현

표준 또는 iframe 코드 구현은 당연히 Brightcove Player를 iframe에 배치 한 다음 HTML 페이지에 배치합니다. 실제 iframe 자체의 스타일을 지정하거나 iframe이 포함 된 페이지의 스타일을 사용하여 크기를 변경할 수 있지만 iframe 페이지의 스타일은 iframe 자체에 계단식으로 캐스케이드하지 마십시오. iframe은 문자 그대로 별도의 HTML 페이지를 가져옵니다. 따라서 iframe의 내용에 대해 원하는 스타일은 iframe의 HTML 페이지와 연결되어야합니다.

브라이트코브 플레이어의 iframe 구현을 사용하는 경우 CSS를 적용하는 단계는 다음과 같습니다.

  • 인터넷에 액세스할 수 있는 파일에 스타일 배치
  • 스타일시트를 플레이어에 연결 (Studio 또는 플레이어 관리 API를 사용하여 완료)

참고: 다음 단계에서는 이전 단계에서 완료한 HTML로 작업하고 위의 코드 조각에 표시된 것으로 가정합니다. 이를 실제 예제로 유지하려면 코드를 다른 파일에 복사하고 새 파일에서 다음 단계를 시작하십시오.

  1. Studio에서 비디오를 게시하고 플레이어의 iframe 구현을 위한 코드를 복사합니다.
  2. HTML 페이지에서 페이지 내 포함 코드를 iframe 코드로 바꿉니다.
  3. HTML 페이지가 다음과 유사한지 확인합니다. 
    <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js .vjs-big-play-button {
      background-color: red;
      opacity: .3;
    }
  </style>
</head>

<body>

<iframe src="https://players.brightcove.net/1507807800001/default_default/index.html?videoId=6065136830001"
  allowfullscreen=""
  allow="encrypted-media"
  width="640" height="360"></iframe>

</body>

</html>
  4. 페이지를 탐색하면 버튼 스타일이 사용되지 않는 것을 볼 수 있습니다.
  5. customize-qs.css파일을 만들고 인터넷에 액세스할 수 있는 위치에 저장합니다.
  6. 버튼의 스타일을 잘라서 파일에 붙여넣고 저장합니다. 버튼의 스타일 만 잘라내십시오. <style>태그. 파일에는 다음 내용만 포함되어야 합니다. 
    .video-js .vjs-big-play-button {
  background-color: red;
  opacity: .3;
}
  7. HTML 페이지가 다음과 유사한지 확인합니다. 
    <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
</head>

<body>

<iframe src="https://players.brightcove.net/1507807800001/default_default/index.html?videoId=6065136830001"
  allowfullscreen=""
  allow="encrypted-media"
  width="640" height="360"></iframe>

</body>

</html>
  8. Studio에서선수모듈을 열고 이 빠른 시작에 사용 중인 플레이어를 편집합니다.
  9. 딸깍 하는 소리플러그인왼쪽 탐색 메뉴에서
  10. 딸깍 하는 소리스타일 시트 , 다음스타일 시트 추가버튼을 누르고 경로를 입력하십시오.
  11. 저장을클릭하여 변경 사항을 저장합니다. 페이지 오른쪽의 미리보기 플레이어가 업데이트되어 버튼에 적용된 CSS 스타일을 표시합니다.
  12. 플레이어를 게시하려면 [ 게시 및 포함] > [변경 사항 게시] 를 클릭합니다 .
  13. 열린 대화 상자를 닫으려면닫기를 클릭합니다 .
  14. 브라우저로 돌아가서 페이지를 새로 고칩니다. 게시된 변경 사항이 활성화되면 iframe 플레이어에 적용된 스타일이 표시됩니다. 게시된 변경 사항이 표시되기까지 몇 분 정도 걸릴 수 있습니다.
