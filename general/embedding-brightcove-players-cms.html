소개

CMS 사용자가 페이지에 비디오 또는 재생 목록 플레이어를 포함할 때 웹 페이지에서 Brightcove 플레이어의 모양과 느낌을 제어할 수 있도록 Brightcove 플레이어 포함 코드를 생성할 때 가능한 한 많은 유연성을 제공하려고 합니다. , 그러나 그들이 HTML 또는 JavaScript 전문가가 될 것이라고 기대하지는 마십시오.

이 항목에서는 이를 수행하는 방법에 대한 지침을 제공합니다. 이 문서의 대상은 주로 Brightcove 파트너이지만 Video Cloud 또는 독립 실행형 Brightcove Player를 CMS와 통합하는 모든 사람에게 유용할 것입니다.

플레이어 포함 대화 상자 동작

사용자가 웹 페이지에서 플레이어의 형식을 제어할 수 있는 대화 상자가 있어야 합니다. 이 대화 상자에서 사용자는 아래 나열된 필드를 설정할 수 있습니다. 이 대화 상자가 표시되기 전에 사용자가 이미 Brightcove 계정과 비디오 또는 재생 목록을 선택했다고 가정합니다. 그렇지 않으면 Brightcove 계정과 비디오 또는 재생 목록을 에 설명된 대로 먼저 선택해야 합니다. 부록아래에.

  1. 플레이어 : 사용자는 사용 가능한 Brightcove 플레이어 중에서 플레이어를 선택할 수 있어야 합니다. 비디오 플레이어를 포함하는 경우 재생 목록이 활성화되지 않은 플레이어만 선택할 수 있습니다. 재생 목록 플레이어를 포함하는 경우 재생 목록 활성화 플레이어만 선택할 수 있어야 합니다.
  2. 자동 재생 : 사용자는 자동 재생을 활성화/비활성화할 수 있어야 합니다. 기본값은 비활성화되어 있습니다.
  3. 음소거 : 사용자는 음소거를 활성화/비활성화할 수 있어야 합니다. 기본값은 비활성화되어 있습니다.
  4. 임베드 유형 : 사용자는 iFrame과 Javascript 포함 코드 중에서 선택할 수 있어야 합니다.
    1. 비디오 플레이어의 경우 기본값은 Javascript여야 합니다.
    2. 재생 목록 플레이어의 경우 기본값은 iFrame이어야 합니다.
  5. 크기 조정 : 사용자는 반응형 중에서 선택할 수 있어야 합니다.[1-1] 또는 고정 크기[1-2] .
    1. 비디오 플레이어의 경우 기본값은 반응형이어야 합니다.
    2. 재생 목록 플레이어의 경우:
      1. iFrame인 경우 기본값은 반응형이어야 합니다.
      2. Javascript인 경우 기본값은 고정이어야 합니다.
  6. 가로 세로 비율 : 사용자는 가로 세로 비율을 선택할 수 있어야 합니다.[1-1] .
    1. 선택 사항은 다음과 같습니다.
      • 16:9
      • 4:3
      • Custom
    2. 기본값은 다음과 같아야 합니다. 16:9 .
  7. 폭, 높이 : 사용자가 너비와 높이를 입력할 수 있어야 합니다.[1-1] .
    1. 가로 세로 비율이 16:9또는 4:3 , NS 키필드는 회색으로 표시되어야 하며 다음을 기반으로 자동 계산됩니다. 너비그리고종횡비 .
    2. 기본값은 다음과 같아야 합니다. 640 ~을위한너비그리고360 ~을위한키가로 세로 비율이 사용자 지정일 때.

노트

  • [1-1] CMS가 사용자가 반응형 대 고정형 및 너비/높이를 지정할 수 있는 코드를 포함하기 위한 기본 컨테이너를 제공하는 경우 크기 조정을 위해 컨테이너를 사용하고 Brightcove 포함 코드가 항상 반응하도록 하는 것이 더 합리적일 수 있습니다.
  • [1-2] 아래에 포함된 반응형 임베드 코드는 너비를 최대 너비로 사용합니다.

매개변수 및 코드 삽입

아래 섹션에서는 사용자 선택을 기반으로 Brightcove 플레이어 내장 코드를 생성하는 방법을 설명합니다. NS %XYZ%값은 아래에 설명된 대로 내장 코드로 대체됩니다.

공통 매개변수

  • %AUTOPLAY% = 자동 재생Autoplay활성화되어 있지 않으면null끈
  • %MUTED% = 음소거된 경우Muted활성화되어 있지 않으면null끈
  • %ACCOUNTID% = 사용자가 선택한 Brightcove 계정 ID
  • %VIDEOID% = 사용자가 선택한 비디오 ID
  • %PLAYLISTID% = 사용자가 선택한 재생 목록 ID
  • %PLAYERID% = 사용자가 선택한 비디오 플레이어 ID 또는 재생 목록 플레이어 ID
  • %CMS% = CMS의 이름
  • %CMSVERSION% = CMS 버전
  • %CONNECTORVERSION% = 커넥터 버전

부록

플레이어 삽입 대화 상자를 열기 전에 비디오 또는 재생 목록을 선택하지 않은 경우 대화 상자에서 비디오 또는 재생 목록(및 필요한 경우 Brightcove 계정)을 선택할 수 있어야 합니다. 대화 상자는 아래 나열된 필드를 사용자에게 표시해야 합니다.

  1. 브라이트코브 계정: 아직 선택하지 않은 경우 사용자는 Brightcove 계정을 선택할 수 있어야 합니다.
  2. 검색 필터: 사용자는 검색 문자열을 입력하여 표시된 비디오 또는 재생 목록의 목록을 필터링할 수 있어야 합니다. Brightcove 검색 API를 사용하는 경우 검색 문자열은 URI로 인코딩되어야 합니다(아래 예 참조).
  3. 폴더: 사용자는 표시된 비디오 목록을 필터링하기 위해 Brightcove 계정에서 폴더 이름을 선택할 수 있어야 합니다. Brightcove API는 폴더에 대한 내장 필터를 제공하지 않으므로 클라이언트 코드는 모든 비디오를 검색한 다음 폴더별로 필터링해야 합니다. (재생 목록에는 적용되지 않음)
  4. 한계: 사용자는 주로 검색 성능을 개선하기 위해 반환되는 동영상 수를 제한할 수 있어야 합니다. Brightcove 검색을 사용할 때 지정할 수 있는 최대 제한은 100입니다. 100개 이상의 비디오를 반환하려면 페이징 메커니즘을 구현해야 합니다. 또한 사용자가 필터링할 폴더를 선택한 경우 클라이언트는 페이징 메커니즘을 사용하여 계정의 모든 비디오를 요청한 다음 폴더별로 해당 목록을 로컬로 필터링하고 사용자 제한에 지정된 비디오 수를 반환해야 합니다. (재생 목록에는 적용되지 않음)
  5. 정렬 기준: 사용자는 정렬 필드를 선택할 수 있어야 합니다.
    1. 동영상의 경우 사용자는 동영상 이름, 업데이트 날짜, 생성 날짜, 시작 날짜 및 총 재생 횟수를 선택할 수 있어야 합니다. 기본값은 업데이트된 날짜여야 합니다.
    2. 재생 목록의 경우 사용자는 이름과 수정 날짜를 선택합니다. 기본값은 수정된 날짜여야 합니다.
  6. 정렬 순서: 사용자는 오름차순 또는 내림차순 정렬 순서를 선택할 수 있어야 합니다. 기본값은 내림차순이어야 합니다.

대화 상자의 동작은 다음과 같아야 합니다.

  1. 위의 사용자 선택을 기반으로 비디오 또는 재생 목록을 표시합니다. 사용자가 하나의 비디오 또는 하나의 재생 목록을 선택할 수 있습니다.
  2. 비디오 목록을 표시할 때:
    1. 활성 비디오만 나열되어야 합니다.
    2. 썸네일 이미지, 동영상 이름 및 ID가 표시되어야 합니다.
    3. 하나의 비디오만 선택할 수 있습니다.
  3. 재생 목록을 표시할 때:
    1. 재생 목록이 수동 재생 목록인 경우 재생 목록 이름, ID 및 재생 목록의 동영상 수를 표시합니다.
    2. 재생 목록이 스마트 재생 목록인 경우 동영상 개수 대신 재생 목록 이름, ID 및 -를 표시합니다.
    3. 하나의 재생 목록만 선택할 수 있습니다.
  4. 새 브라우저 탭에서 동영상 또는 재생목록 플레이어를 여는 클릭 가능한 링크를 표시합니다.

    반환된 비디오는 다음과 같은 완전한 문구가 포함된 모든 것입니다. great blue[2-1]
