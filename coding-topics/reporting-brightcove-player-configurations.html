자격 증명 가져오기

이 구성보고 앱을 사용하려면 적절한 자격 증명이 필요합니다.

대부분의 경우 자격 증명을 얻는 가장 쉬운 방법은 Studio Admin API 인증 섹션 (계정에 대한 관리자 권한 필요) 을 사용하는 것입니다. 자세한 내용은 API 인증 자격 증명관리를 참조하십시오. 다음과 같은 단일 권한이 부여됩니다.

필요한 권한을 Studio에서 사용할 수 없거나 OAuth API에서 직접 가져오려는 경우 아래 나열된 클라이언트 자격 증명가져오기문서를 선택하십시오. 어떤 옵션을 선택하든 올바른 작업 권한을 요청해야 합니다. 다음을 cURL 또는 Postman과 함께 사용하여 Player Management API에 대한 모든 권한을 얻을 수 있습니다.

  "operations": [
    "video-cloud/players/read"
  ]
  • OAuth: cURL을 사용하여 클라이언트 자격 증명 가져오기
  • OAuth: Postman을 사용하여 클라이언트 자격 증명 가져오기

코드펜

이 GitHub 저장소에서 이 샘플과 관련된 모든 코드를 찾습니다 .
보고 싶은 다른 구성 보고서가 있으면 이메일을 보내주십시오. 지식 팀그 요청과 함께.

다음 사항에 유의하십시오.

  • 계정의 플레이어에로드 된 모든 플러그인을 표시하는 보고서는 버전 5 이하 플레이어에서 작동하지 않습니다.
  • 또한 플러그인 보고서에서 다음으로 시작하는 플러그인은 @ brightcove / Brightcove에서 제공하는 플러그인입니다. 나머지는 사용자 정의 플러그인이거나 Studio 또는 curl을 통해 수동으로로드 된 Brightcove 제공 플러그인입니다.

펜보기 Brightcove Player 필터링 된 보고서게시자 : Brightcove Learning Services ( 뿡 빵뀨 ) 의 위에 CodePen .

  • 딸깍 하는 소리 CodePen에서 편집이 CodePen을 자신의 계정으로 포크하려면 오른쪽 상단 모서리에 있습니다.

프록시 코드

이 페이지에서 샘플 앱을 자체 버전을 빌드하려면 자체 프록시를 만들고 호스팅해야 합니다. (브라이트코브 학습 서비스에서 사용하는 프록시는 브라이트코브 도메인의 요청만 허용합니다.) 샘플 프록시는 우리가 사용하는 것과 매우 유사하지만 브라이트코브 도메인이 아닌 도메인의 요청을 차단하는 검사가 없으면이 GitHub 저장소에서 찾을 수있습니다. 또한 여기에서 사용에 대한 기본 지침과 프록시 주변에서 앱을 빌드하는 더 정교한 가이드를 찾을 수 있습니다. REST API 사용 .
