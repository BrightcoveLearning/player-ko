PIP(Picture-in-Picture) 플러그인 변경 로그

다음 트랙은 플러그인에 대한 변경 사항입니다.

변경 내역

21년 4월 20일

v1.1.3

  • 전체 화면 모드에서 나온 후 PiP 오버레이가 작동하지 않는 문제를 해결했습니다.

2020 년 1 월 29 일

v1.1.2

  • 페이지 내 PiP가 루프에 갇혀 플레이어가 깜박이는 문제를 수정했습니다.

2019년 12월 3일

v1.1.1

  • 페이지 내 PIP가 루프에 멈춰 플레이어가 깜박이는 문제를 수정했습니다.

2019년 2월 20일

v1.1.0

  • 향후 Studio 기반 반응 형 소스 코드에 대한 지원을 추가합니다. Studio에서 제공하는 반응 형 소스 코드가 곧 개선 될 예정입니다. 이러한 변경 사항은 PIP 플러그인이 해당 변경 사항이 적용될 때 이러한 임베드 코드를 올바르게 지원하도록 보장합니다.

2018 년 12 월 4 일

v1.0.4

  • 버그 수정: 번들에 loscore를 포함하지 마십시오.
  • 버그 수정: allowOnMobile이 존중되지 않는 문제를 수정했습니다.
  • 버그 수정: 설치 문제 방지를 위해 사후 설치 스크립트 제거
  • 플러그인 생성기 v7.2.0을 사용하여 업데이트

2018년 8월 3일

v1.0.3

  • 버그 수정: 플러그인 인스턴스화시보기 가능 옵션 설정

2018년 6월 11일

v1.0.2

  • 늦게 초기화 된 PIP 플레이어가 초기화시 PIP 동작을 트리거 할 수있는 드문 경쟁 조건을 수정합니다. 이제 PIP 모드를 활성화하기 전에 사용자 활동을 기다립니다.

2018 년 5 월 25 일

v1.0.1

  • 반응 형 플레이어를위한 고정 Picture-in-Picture 모드

2018 년 3 월 26 일

v1.0.0

  • 플레이어가 페이지에서 분리되고 뷰포트에 고정되는 PIP 모드로 전환 할 수 있습니다.
