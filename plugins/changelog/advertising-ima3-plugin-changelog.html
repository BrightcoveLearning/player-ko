    IMA3 플러그인 변경 로그를 사용한 광고

    다음은 플러그인에 대한 변경 사항을 추적합니다.

    변경로그

    분류: 8월 10일

    v3.6.0

    • 컨트리브 광고 6.7.0으로 업그레이드됨

    분류: 7월 13일

    v3.5.0

    • 기능: 플러그인이 등록되지 않은 경우 미들웨어를 사용하지 않게하십시오.
    • 버그 수정: 클릭시 프리 롤 오류 수정다음재생 목록 플레이어의 버튼

    분류: 4월 16일

    v3.4.4

    • 결정된광고 선택오버레이가 표시되지 않음

    분류: 3월 3일

    v3.4.3

    • 버그 수정: 실시간 프리롤 광고 중에 기록되는 가끔 발생하는 오류 해결
    • 되돌리기: 새asyncAutoplaySupport옵션으로 자동 재생 기능 감지가 해결될 때까지 IMA3 초기화 지연 허용

    분류: 1월 7일

    v3.4.2

    • Video.js 7.7.x와의 호환성 수정

    2019년12월20

    v3.4.1

    • 리팩터링: 향후 버전의 Video.js 호환되지 않도록 AdPlayer의 구현을 조정하십시오.

    3일 (12월)

    v3.4.0

    • 새asyncAutoplaySupport옵션으로 자동 재생 기능 감지가 해결될 때까지 IMA3 초기화 지연 허용
    • 음소거된 자동 재생이 iOS 기기에서 백그라운드 오디오를 중지하지 않도록 방지
    • 구현 조정AdPlayer Video.js의 향후 버전과 잠재적 인 비 호환성 방지

    2019년 9월25

    v3.3.2

    • 확인하여 경쟁 조건을 수정했습니다. inAdBreak소스가 전환 될 때의 기능입니다

    2019년 9월9일

    v3.3.1

    • 버그 수정: IMA 플러그인에서 오류를 발생시키는 브라이트코브 플레이어 6.36과 호환되지 않던 문제를 수정했습니다.
    • 버그 수정: 플레이어 크기 조정 이벤트에서 ADSManager 크기 조정

    분류: 8월 5일

    v3.3.0

    • IMA SDK 노출ima3-ready요청 모드 일 때 이벤트 객체ondemand
    • 광고 볼륨이 음소거 처리로 잘못 해석되던 문제를 수정했습니다.
    • 누락과 관련하여 가끔 발생하는 오류 수정getRemainingTime특성

    2019년 5월

    v3.2.6

    • 업그레이드 videojs-contrib-ads 6.6.3으로

    2013월 21일

    v3.2.5

    • 버그 수정: ima3 광고 오류 이후 자동 재생이 차단되던 문제를 수정했습니다.
    • 버그 수정: 실시간 광고 삽입을 사용하는 경우 프리롤을 기다리는 동안 재생을 차단하지 마십시오.

    2019년 1월

    v3.2.4

    • Studio를 통해 추가 된 미디어 큐 포인트 키 / 값 메타 데이터가 검색어 매개 변수로 광고 태그에 추가되는 버그를 수정mediainfo.ad_keys매크로

    2013년 1월 3일

    v3.2.3

    • 주로 서비스 업데이트
    • 버그 수정: 이제 Video Cloud를 통해 설정된 광고 큐포인트의 정확도가 밀리초 단위로 올바르게 작동합니다.

    10월 24일

    v3.2.2

    • 호출 할 버그 수정adMacroReplacement언제serverUrl빈 문자열
    • 오버레이 광고로 마우스를 올리면 IE/가장자리 컨트롤 바 가시성을 허용하는 문제가 해결되었습니다.

      • 페이지 변수 매크로가있는 URL은 다음과 같습니다. http://example.com?foo={pageVariable.foo} (해당 매크로는 변수 값으로 대체됩니다. foo광고 요청시 브라우저의 자바 스크립트 컨텍스트에서)
      • 페이지 변수를 아직 사용할 수 없는 경우 광고 요청이 이루어지면 매크로가 작동하지 않습니다.
      • 여기에서 새 광고 매크로 기능에 대한 기술적 인 세부 정보를 확인할 수 있습니다. https://github.com/videojs/videojs-contrib-ads#macros
    • 이제 사용try/catch광고 관리자가 아직 생성되지 않은 경우 폐기 할 때
      • 이것은 엣지 케이스에서 추가적인 안전을위한 것입니다. 차이점을 기대해서는 안됩니다.
