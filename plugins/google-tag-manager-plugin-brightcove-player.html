개요

Google 태그 관리자 (GTM) 플러그인을 사용하면 GTM 기능을 Brightcove Player와 통합 할 수 있습니다.

태그 관리자를 사용하면 코드를 사용하지 않고도 태그를 추가하고 게시할 수 있습니다. 태그 관리자는 클라이언트측 솔루션이므로 다양한 마케팅 기술에 걸쳐 데이터를 수집할 수 있습니다. 이를 통해 마케터는 다양한 수집 지점 집합에서 사용자 및 플레이어 데이터를 수집할 수 있는 능력을 부여합니다.

이미 Google Analytics, Adwords 및 DoubleClick을 사용하고있는 경우 이러한 도구와 잘 통합되므로 Google 태그 관리자를 사용할 수도 있습니다. GTM은 다양한 변수 및 트리거 옵션을 제공합니다.

Google 태그 관리자 플러그인을 사용하면 Google Analytics 대시보드를 사용하여 Brightcove Player 이벤트를 추적할 수 있습니다. 다음 플레이어 이벤트를 추적할 수 있습니다. 

  • 플레이어로드
  • 동영상 노출
  • 동영상 보기
  • 비디오 참여
  • 플레이 요청
  • 동영상 완료
  • 광고 시작, 광고 종료와 같은 고급 플레이어 이벤트
  • 재생 경험을 방해하는 오류
Google 태그 관리자 플러그인은 Brightcove Player 버전 6.36.5 이상에서 지원됩니다.

태그 관리 용어

다음은 Google 태그 관리자 (GTM)를 사용할 때 접하게되는 몇 가지 일반적인 용어입니다.

학기 설명
컨테이너 태그 관리 시스템 계정에는 고유한 태그와 변수가 있는 여러 컨테이너 (프로필) 가 있을 수 있습니다. 각 컨테이너에는 일반적으로 개발, 스테이징 및 프로덕션 용으로 설정된 여러 환경이있을 수 있습니다.
데이터 레이어 데이터 영역은 웹 페이지의 코드에있는 데이터 모음을 나타냅니다. 이 JavaScript 객체는 웹 사이트에서 태그 관리자 컨테이너로 정보를 전달하는 데 사용됩니다. 그런 다음이 정보를 사용하여 태그 구성에서 변수를 채우고 트리거를 활성화 할 수 있습니다.
이벤트 이벤트는 보고 도구로 데이터를 전송하는 데 사용되는 데이터 포인트입니다. 이는 비디오 로딩 및 재생과 관련이 있습니다.
태그 태그는 사용자 상호 작용 데이터를 수집하기 위해 웹 페이지에 배치된 코드 조각입니다. 태그 관리 시스템은 사이트에 태그를 추가하는 프로세스를 단순화하므로 태그 ID 및 태그 이벤트 유형만 제공해야 합니다.
태그 마켓플레이스 태그 마켓플레이스 템플릿은 태그 배포를 위한 코드 스니펫을 제공합니다. 이렇게 하면 필요한 코드 지식의 양이 줄어들지만 성공적인 구현을 위해서는 JavaScript 및 HTML에 익숙해야 합니다.
트래커 추적기는 데이터를 수집, 저장 및 Google 태그 관리자로 전송하는 개체입니다.
방아쇠 트리거는 페이지에 태그 코드를 추가해야하는시기를 알려줍니다. 이러한 조건은 사용자 정의 조건입니다.

시작하기

Google 태그 관리자를 설정하려면 다음 단계를 따르세요.

단계

  1. Google 문서를 검토하여태그 관리자 설정 및 설치 .

  2. 에태그 관리자 , 계정을 만들거나 기존 계정을 사용합니다. 새 계정을 만들려면계정 만들기단추.

    기본적으로 새 계정에 대해 새 컨테이너가 생성됩니다. 각 계정 내에 추가 컨테이너를 만들 수 있습니다.

  3. 계정 이름 및 컨테이너 이름을 포함하여 새 계정 페이지의 값을 입력하십시오. 대상 플랫폼을 선택하십시오.

  4. 선택창조하다단추.

추적 할 이벤트 검토

아래 표를 검토하고 추적 할 이벤트를 결정하십시오.

유형 이벤트 설명
플레이어 로드 bcvideo_player_load 플레이어가로드되었으며 재생할 준비가되었습니다.
비디오 인상 bcvideo_video_impression 동영상이 플레이어에로드되었으며 재생할 준비가되었습니다.
비디오보기 bcvideo_video_view 자동 재생을 통해 또는 사용자 상호 작용으로 인해 동영상이 재생되기 시작했습니다.
비디오 참여 bcvideo_video_engagement 사용자가 동영상 타임라인의 초 범위를 시청했습니다. 이 이벤트는 비디오 참여를 추적하기 위한 하트비트로, 사용자의 상호 작용 및 동영상 길이에 따라 재생 중에 여러 번 전송될 수 있습니다.
플레이 요청 bcvideo_play_request 재생은 사용자가 명시 적으로 재생 버튼을 클릭하거나 플랫폼이 자동 재생 시나리오에서 재생을 트리거 할 때 자동으로 시작됩니다.
광고 시작 bcvideo_ad_start 제어는 재생 플랫폼에 의해 광고 에이전트에게 넘겨집니다.
광고 엔드 bcvideo_ad_end 제어권은 광고 에이전트에서 재생 플랫폼으로 넘겨집니다.
오류 bcvideo_error 재생 경험을 방해하는 치명적인 오류가 발생했습니다.
비디오 완료 bcvideo_video_complete 비디오 재생 완료

변수, 태그 및 트리거 생성

추적하려는 모든 Brightcove 이벤트는 Google 태그 관리자 (GTM)에서 이벤트 태그로 생성되어야합니다. 그러면 GTM의 이벤트 ID가 플레이어 모듈 내에서 사용됩니다. 이를 통해 Brightcove 이벤트 데이터를 GTM 이벤트로 전달할 수 있습니다.

각 이벤트 태그에 대해 이벤트를 Google Analytics로 보내려면 GTM 트리거를 만들어야합니다. 그런 다음 GTM에서변수탭에서 각 이벤트와 관련된 맞춤 매개 변수를 만듭니다.

GTM에 변수 추가

Brightcove 매개 변수는 추가 데이터를 전달하며 이벤트와 함께 전달됩니다. 기록 될 모든 Brightcove 매개 변수는 Google 태그 관리자에서 변수로 생성되어야합니다. 매개 변수 이름은 플레이어 내에서 사용되어 Brightcove 매개 변수가 Google 태그 관리자 변수에 전달 될 수 있습니다.

Google 태그 관리자에서 변수를 만들려면 다음 단계를 따르세요.

  1. Google 태그 관리자 계정에 로그인

  2. 고르다변수왼쪽 탐색에서.

  3. 에서사용자 정의 변수섹션에서새로운단추.

  4. 에 대한가변형 , 고르다데이터 영역 변수 .

  5. 세부 사항 입력 :

    • 데이터 영역 변수 이름 -변수 이름
    • 데이터 계층 버전 - 버전 2

  6. 저장을 선택합니다 .

  7. GTM 변수로 생성 할 수있는 Brightcove 매개 변수 :

    • 계정 -Brightcove 계정 이름
    • 플레이어 -플레이어 ID
    • 선수 이름 - 선수 이름
    • 플랫폼 버전 -플랫폼 버전 번호
    • 세션 -세션 식별자
    • 비디오 -비디오 ID
    • 비디오 이름 -동영상 이름
    • 범위 -StartSecond..EndSecond 형식으로 표시된 초 범위
    • 동영상 조회 시간 (초) -시청 한 초 수
    • 비디오 길이 -동영상 길이 (초)
    • 비디오 조회 율 -비디오 시청 비율
    • 에러 코드 -이벤트와 관련된 플랫폼 별 오류 코드
    • 비디오 마일스톤 -동영상으로 완성 된 마일스톤 (25 %, 50 %, 75 %, 100 %)
    • 비디오 플레이 헤드 -동영상에서 이벤트가 트리거 된 시간
    • 국가 - ISO-3166(알파 2) 국가 코드, 예: 우리
    • 나라 이름 - 사람이 읽을 수 있는 국가 이름 예: 미국
    • 부위 -ISO-3166 (알파 2) 지역 코드
    • 지역 명 -사람이 읽을 수있는 영역 이름
    • 시티 - 도시 이름
    • 브라우저 -사용 된 브라우저의 전체 이름 및 버전
    • 브라우저 유형 -사용 된 브라우저의 일반 이름 (예 : Chrome, Edge, Firefox)
    • 목적지 -이벤트가 발생한 URI
    • 출처 -최종 사용자를 목적지로 보낸 URI
    • 기기 종류 -User-Agent가 신뢰할 수없는 경우 이벤트를 발생시킨 기기의 OS를 지정하도록 재정의
    • 장치 OS -기기에서 사용중인 OS의 이름
    • 기기 OS 버전 -기기에서 사용중인 OS 버전

  8. 사용자 정의 변수는 GTM에서 다음과 유사해야합니다.

GTM에 태그 추가

추적 할 이벤트를 결정한 후 Google 태그 관리자에서 태그를 만들어야합니다.

Google 태그 관리자에서 태그를 만들려면 다음 단계를 따르세요.

  1. Google 태그 관리자 계정에 로그인

  2. 고르다태그왼쪽 탐색 메뉴에서새로운단추.

  3. 선택태그 유형같이 Google Analytics : 유니버설 애널리틱스 .

  4. 태그 세부 정보 입력 :

    • 트랙 유형 - 로 설정행사
    • 범주 -이벤트 카테고리 변수
    • 동작 -이벤트 액션 변수
    • 상표 -이벤트 라벨 변수
    • 값 -없음
    • 비 상호 작용 적중 - 로 설정진실
    • Google Analytics 설정 - 기본 설정
    • 추적 ID -GA 대시 보드의 추적 ID. 추적 ID 변수를 선택합니다.

  5. 고급 설정으로 이동하여 관련 맞춤 측정 기준 색인 및 측정 기준 값을 설정하세요. 이러한 인덱스는 Google Analytics에있는 것과 일치해야합니다.

  6. 태그는 GTM에서 다음과 유사해야합니다.

GTM에 트리거 추가

추적하려는 각 이벤트에 대해 Google 태그 관리자에도 트리거가 필요합니다.

Google 태그 관리자에서 트리거를 만들려면 다음 단계를 따르세요.

  1. Google 태그 관리자 계정에 로그인

  2. 고르다트리거왼쪽 탐색 메뉴에서새로운단추.

  3. 선택트리거 유형같이맞춤 이벤트 .

  4. 이벤트 이름을 입력하십시오.

  5. 추적하려는 각 이벤트에 대한 트리거를 만듭니다.

  6. 트리거는 GTM에서 다음과 유사해야합니다.

Brightcove와 통합

GTM을 Brightcove Player와 통합하려면 Google 태그 관리자 플러그인을 플레이어에 추가하십시오.

스튜디오 사용하기

다음 단계에서는 Video Cloud Studio를 사용하여 플러그인을 로드하는 방법을 자세히 설명합니다.

  1. Video Cloud Studio에서선수모듈을 열고 새 플레이어를 만들거나 플러그인을 추가하려는 플레이어를 찾습니다.
  2. 플레이어의 속성을 열려면 플레이어의 링크를 선택합니다.
  3. 선택하다플러그인왼쪽 탐색 메뉴에서

  4. 그런 다음 플러그인추가버튼을 선택한 다음브라이트코브 플러그인을 선택합니다 .

  5. 확장 Brightcove 플러그인드롭 다운 및 선택 Google 태그 관리자 .

  6. 플레이어에게 추적하려는 이벤트 및 맞춤 측정 기준을 알리려면 JSON 개체를 만들어옵션 (JSON)텍스트 상자.

    필드 설명:

    • tracker -Google 태그 관리자 추적 ID (GTM-XXXXXXX)
    • eventsToTrack -추적하려는 이벤트
    • paramsToTrack -추적하려는 맞춤 측정 기준

    다음은 예제입니다.

    {
  "tracker":"your Google Tag Manager id",
	"paramsToTrack": {
    "bcvideo_video_seconds_viewed":"video_seconds_viewed",
    "bcvideo_video_playhead":"video_playhead",
    "bcvideo_account":"account",
    "bcvideo_player":"player",
    "bcvideo_player_name":"player_name",
    "bcvideo_video":"video",
    "bcvideo_video_name":"video_name",
	"bcvideo_session":"session",
    "bcvideo_platform_version":"platform_version",
	"bcvideo_range":"range",
    "bcvideo_video_duration":"video_duration",
    "bcvideo_video_percent_viewed":"video_percent_viewed",
    "bcvideo_error_code":"error_code",
    "bcvideo_video_milestone":"video_milestone",
    "bcvideo_destination":"destination"
  },
  "eventsToTrack": {
    "video_impression": "Video Impression",
	"video_view": "Video View",
	"video_complete": "Video Complete",
    "play_request": "Play Request",
    "video_engagement": "Video Engagement",
    "ad_start": "Ad Start",
    "ad_end": "Ad End",
    "player_load": "Player Load",
    "error": "Error"
  }
}
    JSON 개체의 맞춤 측정기준이 Google 애널리틱스의 이름 및 색인과 일치하는지 확인하세요.

  7. 저장버튼을 선택합니다. 이제 Google 태그 관리자플레이어의 플러그인 목록에 플러그인이 추가되었습니다.

  8. 플레이어를 게시하려면 [ 게시 및 포함] > [변경 사항 게시] 를 선택합니다 .
  9. 대화상자를 닫으려면닫기를 선택합니다 .

  10. 돌아 가기미디어방금 Google 태그 관리자 용으로 업데이트 한 플레이어를 사용하여 동영상이나 재생 목록을 게시하세요.

    플레이어가 로드되고 사용자가 플레이어와 상호 작용하면 지정된 이벤트와 파라미터가 추적됩니다.

맞춤 측정 기준 만들기

맞춤 측정 기준은 Google 애널리틱스에서 직접 만드는 측정 항목입니다. 자세한 내용은 Google의맞춤 측정 기준 및 측정 항목문서.

Google Scripts를 사용하면 Google Analytics에서 맞춤 측정 기준을 추가하는 프로세스를 자동화 할 수 있습니다.

단계

Google App Script를 사용하여 맞춤 측정 기준을 만들려면 다음 단계를 따르세요.

  1. Google에 로그인하십시오.

  2. 다음으로 이동 G Suite 개발자 허브 Apps Script 새 빈 프로젝트를 선택합니다.

  3. 고르다제목없는 프로젝트프로젝트 이름을 지정하십시오.

  4. 다음 사용자 지정 앱 스크립트 코드를 편집기에 복사하고 다음 값을 업데이트합니다.

    • accountId -Google Analytics 계정 ID
    • webPropertyId -Google 계정 추적 ID

동영상 시청률 ( video_engagement )는 다음과 같이 계산할 수 있습니다 (Google 및 Adobe 분석의 video_engagement 이벤트와 유사). 

Video watched percentage = duration the user actually watched / total video duration

예를 들어 20초 비디오에서 아래 두 가지 경우 모두 25% 이정표를 달성할 수 있습니다.

  • 0-5초에서 사용자 시계
  • 사용자 시계는 0-1초에서 11초로 건너뛰고 15초로 시청합니다.

위의 두 경우 모두 사용자가 5초를 시청했기 때문에 이정표가 25% 로 보고됩니다.
