플러그인 레지스트리란 무엇입니까?

플러그인 레지스트리는 Brightcove 플러그인에 대한 정보를 저장하는 데 사용되는 Player Management API 섹션입니다. 레지스트리는 저장된 각 플러그인의 모든 버전과 관련 스크립트 및 스타일시트에 대한 참조 역할을 합니다. 플레이어 구성은 플러그인 레지스트리를 활용하여 현재 플러그인 버전을 표시하고 플러그인 버전을 지정할 수도 있습니다.

플러그인 레지스트리는 Player Management API 버전 2에서 구현됩니다.

장점

Brightcove Player 사용자를 위한 플러그인 레지스트리의 장점은 다음 세 가지입니다.

  • 플레이어에서 사용 중인 플러그인 버전과 해당 플러그인 버전과 관련된 특정 JavaScript 및 CSS 파일을 정확히 볼 수 있습니다.
  • 플레이어 버전과 호환되는 새 버전이 출시되면 플러그인이 자동으로 업데이트되도록 구성할 수 있습니다.
  • 어떤 이유로 플레이어를 특정 플레이어 버전으로 유지하기로 선택한 경우 플러그인은 잠겨 있을 때 해당 플레이어에 대해 동일한 특정 버전으로 유지됩니다. 플러그인이 플레이어 버전과 호환되지 않는 버전으로 업그레이드되는 것에 대해 걱정할 필요가 없습니다.

플러그인 사용 지정

플러그인 사용을 지정하려면 플레이어 생성 시 원하는 플러그인의 레지스트리 ID를 제공하기만 하면 됩니다. 자동 업데이트되는 주 버전(예: 2.x ) 또는 부 버전(예: 2.0.10 ) 자동 업데이트되지 않습니다. 버전 설정에 대한 자세한 내용은 이 문서의 뒷부분에 나와 있습니다. 당신은 또한 제공할 수 있습니다options플러그인의 경우.

    curl \
      --header "Content-Type: application/json" \
      --user $EMAIL \
      --request POST \
      --data '{
        "name": "Plugin Registry Test Player",
        "configuration": {
          "video_cloud": {
            "video": "4093372393001"
          },
          "plugins": [
           {
              "registry_id": "@brightcove/videojs-custom-endscreen",
              "version": "2.x",
              "options": { "content": "<strong>Content</strong> for <em>custom</em> end screen" }
           }
          ]
        }
      }' \
      https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players

특정 주 버전 지정

특정 플러그인에 대해 최신 버전의 플러그인을 사용하도록 플레이어를 구성할 수 있습니다. 주요한사용 중인 플레이어와 호환되는 버전입니다. 이 작업을 수행하고 플러그인의 마이너 버전이 릴리스되면 플러그인은 자동으로 최신 마이너 버전으로 업데이트됩니다.

예를 들어 플레이어에 호환되는 최신 버전의 사용자 정의 최종 화면 플러그인이 포함되도록 하려면메이저 버전 2플러그인의 다음을 사용합니다:

    curl \
      --header "Content-Type: application/json" \
      --user $EMAIL \
      --request POST \
      --data '{
        "name": "Plugin Registry Test Player",
        "configuration": {
          "video_cloud": {
            "video": "4093372393001"
          },
          "plugins": [
            {
              "registry_id": "@brightcove/videojs-custom-endscreen",
              "version": "2.x"
            }
          ]
        }
      }' \
      https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players

결과 구성은 다음과 유사하게 나타납니다.

    "plugins": [{
      "injected_version": "2.0.10",
      "name": "customEndscreen",
      "registry_id": "@brightcove/videojs-custom-endscreen",
      "version": "2.x"
    }],
    "scripts": [
      "//players.brightcove.net/videojs-custom-endscreen/2.0.10/videojs-custom-endscreen.min.js"
    ],
    "stylesheets": [
      "//players.brightcove.net/videojs-custom-endscreen/2.0.10/videojs-custom-endscreen.css"
    ],

특정 버전 지정

사용할 플러그인의 특정 버전을 지정할 수 있습니다. 이렇게 하면 어떤 상황에서도 플러그인이 자동으로 업데이트되지 않으므로 필요한 경우 수동으로 업데이트해야 합니다.

에 설명된 대로 플레이어 버전을 잠근 경우플레이어 업데이트 모드 및 버전 설정문서에서 해당 플레이어와 연결된 플러그인도 버전이 잠깁니다.

플러그인 버전을 잠그려면 플레이어를 생성할 때 버전을 지정하기만 하면 됩니다.

    curl \
      --header "Content-Type: application/json" \
      --user $EMAIL \
      --request POST \
      --data '{
        "name": "Plugin Registry Test Player",
        "configuration": {
          "video_cloud": {
            "video": "4093372393001"
          },
          "plugins": [
            {
              "registry_id": "@brightcove/videojs-overlay",
              "version": "1.0.2"
            }
          ]
        }
      }' \
      https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players

플레이어 구성은 특정 버전과 JavaScript 및 CSS 파일을 반영합니다.

    "plugins": [{
      "name": "overlay",
      "registry_id": "@brightcove/videojs-overlay",
      "version": "1.0.2"
    }],
    "scripts": [
      "//players.brightcove.net/videojs-overlay/1.0.2/videojs-overlay.min.js"
    ],
    "stylesheets": [
      "//players.brightcove.net/videojs-overlay/1.0.2/videojs-overlay.css"
    ],

호환되지 않는 할당

호환되지 않는 플러그인을 플레이어에 할당하거나 이전에 할당된 플러그인이 더 이상 호환되지 않도록 플레이어의 버전을 변경하면 오류가 발생합니다. 오류는 여기에 자세히 설명되어 있습니다.

호환되지 않는 플레이어에 플러그인 할당

플레이어 생성 또는 업데이트 중 언제라도 플레이어 버전과 호환되지 않는 플러그인 버전을 지정하려고 하면 오류가 발생합니다. 예를 들어 다음은 호환되지 않는 이전 플러그인을 새 플레이어에 할당하려고 할 때 API의 응답입니다.

    {
      "message": "There are no versions of the plugin that match the specified major version. The major version specified was 1.x for plugin @brightcove/videojs-bc-playlist-ui",
      "error_code": "PLAYER_MANAGEMENT_ERROR",
    }

플러그인과 호환되지 않도록 플레이어 버전 변경

기존 플레이어의 버전을 변경하려고 하고 플레이어에 새로 할당된 플레이어 버전과 호환되지 않는 플러그인이 이미 있는 경우 다음과 유사한 오류가 발생합니다.

    {
      "error_code":"APPLICATION_ERROR",
      "message":"plugin \"@brightcove/videojs-ima3\" is not compatible with template 5.28.1"
    }

레지스트리 정보 업데이트

표준 절차를 사용하여 플레이어의 플러그인 레지스트리 정보를 업데이트할 수 있습니다. 예를 들어 다음 curl 문은 HTTP를 사용합니다. PATCH 특정 버전에서 주요 버전 업데이트 동작으로 플러그인을 변경하는 방법:

    curl \
      --header "Content-Type: application/json" \
      --user $EMAIL \
      --request PATCH \
      --data '{
        "plugins": [{
          "registry_id": "@brightcove/videojs-custom-endscreen",
          "version": "2.x"
        }]
      }' \
      https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration

변경한 후에는 플레이어를 게시해야 합니다.

플러그인 레지스트리 API

다음과 같은 기능을 제공하는 레지스트리에서 플러그인 정보를 검색할 수 있는 두 개의 새로운 끝점이 Player Management API에 추가되었습니다.

  • 레지스트리에서 모든 플러그인을 검색합니다.
  • 특정 플러그인에 대한 정보를 검색합니다.

모든 플러그인 가져오기

다음을 사용하여 현재 레지스트리에 있는 플러그인에 대한 정보를 검색하십시오.

    curl \
      --header "Content-Type: application/json" \
      --user $EMAIL \
      --request GET \
      https://players.api.brightcove.com/v2/plugins
참고: 양식에 쿼리 문자열 추가?template_version=$SVTVERSION 엔드포인트에 대한 응답은 지정된 단일 비디오 템플릿 버전과 호환되는 플러그인 버전만 포함하는 응답을 생성합니다. 예를 들어 Brightcove Player 템플릿 6.7.0과 호환되는 플러그인 버전만 가져오려면 다음을 사용하십시오. ?template_version=6.7.0 .
