출시 전 광고 플러그인 테스트

{{ page.description }}

개요

Brightcove Player에서 비디오 광고를 표시하는 것은 복잡한 작업 일 수 있습니다. 여러 엔티티 (Brightcove Player 자체, 광고 서버 및 특정 광고 플러그인)는 모두 성공적인 광고 전달에 관여합니다. IMA3 및 FreeWheel 플러그인 업데이트를위한 테스트 환경을 제공하기 위해 Brightcove는 플러그인의 출시 전 버전을 제공합니다. 이 문서는 이러한 시험판으로 테스트하는 방법에 대한 지침을 제공합니다.

프리 릴리즈는 Brightcove 웹 플레이어 개발자 포럼 ,뿐만 아니라변경 로그섹션 IMA3 플러그인으로 광고과 FreeWheel 플러그인으로 광고서류.

Studio에서 IMA3 테스트

다음 단계는 Studio에서 출시 전 IMA3 플러그인을 테스트하는 방법을 안내합니다.

  1. 안전을 위해 현재 사용중인 플러그인의 현재 버전 /plugins/plugin-version-reference.html을 기록해야합니다. 이렇게 하면 테스트 전에 사용하던 플러그인 버전을 복원할 수 있습니다. 항상 최신 버전의 플러그인을 찾을 수 있습니다. 플러그인 버전 참조문서.
  2. 열기선수모듈을 열고 새 플레이어를 만들거나 플러그인을 추가하려는 플레이어를 찾습니다.
  3. 플레이어의 속성을 열려면 플레이어 링크를 클릭합니다.
  4. 딸깍 하는 소리광고하는왼쪽 탐색 메뉴에서
  5. 선택 취소클라이언트 측 (IMA) 활성화 . 곧 수동으로로드 할 버전을 방해 할 수 있으므로이 기능을 사용하지 않는 것이 좋습니다.
  6. 저장을 클릭합니다 .
  7. 딸깍 하는 소리플러그인왼쪽 탐색 메뉴에서
  8. 그런 다음 플러그인추가를 클릭합니다 .
  9. 플러그인 이름에 를입력합니다ima3 .
  10. 자바스크립트 URL의 경우 시험판 자바스크립트 파일의 URL을 입력합니다.
  11. CSS URL의 경우 시험판 CSS 파일의 URL을 입력합니다.
  12. 에 구성 옵션을 입력합니다. 옵션(JSON)텍스트 상자. 이전에 다음을 사용하여 IMA3 플러그인을 구성한 경우플러그인섹션에서 옵션을 변경할 필요가 없습니다.
  13. 저장을 클릭합니다 .
  14. 플레이어를 사용하려면 Publish & Embed를 클릭합니다 . 미리 보기 플레이어를사용하여 테스트하거나게시된 플레이어를 게시하고 사용할 수 있습니다. .
  15. 열린 대화 상자를 닫으려면닫기를 클릭합니다 .

Studio에서 FreeWheel 테스트

다음 단계는 Studio에서 시험판 FreeWheel 플러그인을 테스트하는 방법을 안내합니다.

  1. 안전을 위해 현재 사용 중인 플러그인의 최신 버전을 기록해야 합니다. 이렇게 하면 테스트 전에 사용하던 플러그인 버전을 복원할 수 있습니다. 항상 최신 버전의 플러그인을 찾을 수 있습니다. 플러그인 버전 참조문서.
  2. 열기선수모듈을 열고 새 플레이어를 만들거나 플러그인을 추가하려는 플레이어를 찾습니다.
  3. 플레이어의 속성을 열려면 플레이어 링크를 클릭합니다.
  4. 딸깍 하는 소리플러그인왼쪽 탐색 메뉴에서
  5. 다음을 클릭하십시오. 플러그인 추가또는 당신이 본다면 FreeWheelPlugin플러그인 목록에서 클릭하십시오.
  6. 플러그인 이름에 를입력합니다FreeWheelPlugin .
  7. 자바스크립트 URL의 경우 시험판 자바스크립트 파일의 URL을 입력합니다.
  8. CSS URL의 경우 시험판 CSS 파일의 URL을 입력합니다.

  11. 플레이어를 사용하려면 Publish & Embed를 클릭합니다 . 미리 보기 플레이어를사용하여 테스트하거나게시된 플레이어를 게시하고 사용할 수 있습니다. .

프로그래밍 방식 구현 테스트

플러그인을 프로그래밍 방식으로 구현 한 테스트는 IMA3 플러그인을 사용하든 FreeWheel 플러그인을 사용하든 동일합니다. 기본적으로 JavaScript 및 CSS 파일의 기존 경로를 시험판 버전의 경로로 바꾸면됩니다. 이 작업을 수행하는 단계는 다음과 같습니다.

  1. 안전을 위해 현재 사용 중인 플러그인의 최신 버전을 기록해야 합니다. 이렇게 하면 테스트 전에 사용하던 플러그인 버전을 복원할 수 있습니다. 항상 최신 버전의 플러그인을 찾을 수 있습니다. 플러그인 버전 참조문서.
  2. 코드에서<link>과<script>적절한 플러그인에 대한 CSS 및 JavaScript를 가져 오는 태그. 아래 코드에서 IMA3 플러그인의 예가 강조 표시되어 있습니다.
  3. 해당 경로를 시험판 버전으로 바꿉니다.
  4. 페이지를 새로 고치면 시험판 버전을 사용하게됩니다.
  5. 테스트가 완료되면 기록 된 경로를 사용하여 JavaScript 및 CSS 파일에 대한 경로를 다시 편집하여 원래 구성을 복원하십시오.
