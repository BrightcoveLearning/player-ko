비 Brightcove 플레이어 사용

{{ page.description }}

소개

Brightcove 웹 및 모바일 장치 플레이어에는 Video Cloud 계정의 다양한 측면을 작동하기 위해 직접 또는 플러그인을 통해 상당한 양의 기능이 내장되어 있습니다.

  • 비디오 데이터 및 메타 데이터 검색
  • 도메인 또는 지리적 제한을 통한 콘텐츠 보호 처리
  • Brightcove 데이터 수집기로 분석 데이터 보내기

어떤 이유로 Video Cloud 비디오를 재생해야하는 경우이 기능을 직접 처리해야합니다. 아래 섹션에서는 Brightcove API를 사용하여이를 수행하는 방법을 설명합니다.

정책 키 가져오기

비디오 및 재생 목록 데이터를 가져 오거나 콘텐츠 제한을 설정하려면정책 키 . 대부분의 경우 Video Cloud Studio를 통해이 작업을 수행하거나 Policy API를 사용할 수 있습니다.

간단한 정책 키

ID로 동영상 및 / 또는 재생 목록을 검색하기 만하면됩니다. 플레이어로부터 정책 키를 얻다 .

당신이 할 수 있고 싶다면비디오 검색사용재생 API , 검색 가능한 정책 키를 만들어야합니다. 정책 API를 통해이를 수행하거나빠른 시작: 정책 API .

재생 제한이있는 정책 키

플레이어에 대한 도메인 제한 및 지역 제한도 정책 키를 사용하여 구현됩니다. 둘 중 하나 또는 둘 다 필요한 경우 정책 키를 얻는 가장 쉬운 방법은 Studio의 새 플레이어에 원하는 제한을 설정 한 다음그 플레이어로부터 정책 키를 얻습니다. . 원하는 경우 다음을 사용하여이 정책 키를 직접 생성 할 수도 있습니다. 정책 API .

비디오 데이터 가져오기

정책 키가 있으면 다음을 사용하여 플레이어의 비디오 데이터를 검색 할 수 있습니다. 재생 API . 데이터는 세 가지 방법으로 검색 할 수 있습니다.

  • ID 또는 reference_id로 동영상 가져 오기
  • ID 또는 reference_id로 재생 목록 가져 오기
  • 동영상보기검색

참조빠른 시작: 재생 API샘플 JavaScript의 경우 API 요청을 설정하는 데 사용할 수 있습니다.

응답 데이터에는 비디오가 포함됩니다. sources , URL을 포함한 변환 데이터의 배열입니다. 또한 썸네일 및 포스터 이미지에 대한 URL과 비디오와 같은 메타 데이터도 제공됩니다. name과description .

이 데이터를 플레이어에로드하는 방법은 플레이어의 API에 따라 다릅니다.

해석학

비 Brightcove 플레이어는 자동으로 데이터를데이터 수집 API Brightcove Analytics 시스템의 경우이를 구현하는 것은 귀하의 책임입니다.

수행해야 할 작업은 플레이어 용 API, 특히 플레이어 작업을 위해 브로드 캐스트되는 이벤트에 따라 다릅니다. 참조데이터 수집 API 샘플캡처해야하는 이벤트 및 API에 데이터를 제출하는 방법에 대한 지침은 VideoJS 플레이어 기반 모델을 참조하십시오.

기타 기능

광고 또는 DRM 잠금 해제와 같은 다른 기능이 필요한 경우 플레이어에도이를 구현해야합니다. 소스 코드를 살펴보면 Brightcove Player 플러그인해야 할 일에 대한 몇 가지 유용한 단서를 제공합니다.
