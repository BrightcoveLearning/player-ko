표준(iframe) 포함 코드에 사용 가능한 옵션

{{ page.description }}

소개

그만큼 iframe플레이어 구현 코드는 플레이어를 하나의 포함 된 단위로 쉽게 사용할 수 있도록합니다. 이것은 종종 소셜 미디어 응용 프로그램 및 공유에 필요합니다.

사용량

참고: Brightcove Player 고객 인 경우videoId쿼리 매개 변수를 사용하여 플레이어에 동영상을 할당합니다. 플레이어모듈에서 비디오를 할당하거나 curl을 사용합니다.

사용하려면 iframe코드는 다음 단계를 따릅니다.

  1.  PLAYERS모듈을 사용하여 플레이어를 만듭니다.
  2. MEDIA모듈로 이동하여 새로 만든 플레이어를 사용하여 비디오를 게시합니다.
  3. 복사표준 소스 코드. HTML은 다음과 유사합니다. 
          <iframe src="https://players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen></iframe>
  4. 브라우저에서 HTML을 복사하여 전체 HTML 페이지에 붙여body넣습니다.
  5. HTML 페이지를 탐색하여 플레이어 기능을 확인합니다.

쿼리 매개변수

다양한 검색어 매개 변수를 iframe태그를 사용하여 플레이어의 행동 방식에 대한 추가 정보를 제공합니다. 아래 표에는 사용 가능한 매개 변수가 자세히 나와 있습니다.

true 또는 false 만 될 수있는 쿼리 매개 변수를 사용하는 경우 (데이터 유형부울 ) 쿼리 매개 변수를 포함하여 켜거나 제외하여 끄면됩니다. 여기에서는muted .

<iframe src="https://players...index.html?videoId=4784463159001&muted"

  • play : 플레이어는 수동으로 play를 호출하려고 시도합니다.

  • any : 플레이어는 먼저 play를 호출하려고 시도합니다. 실패하면 플레이어를 음소거하고 Play를 호출합니다.
위 중 하나라도 실패하면 플레이어는 자동 재생이 false로 설정된 것처럼 “빅 플레이 버튼”을 표시합니다.

비디오를 자동 재생하도록 하는 것은 복잡한 상황일 수 있습니다. 자세한 내용은자동 실행 고려 사항문서를 참조하십시오.

프리로드 세부 정보

preload 매개 변수는 동영상 태그가로드되는 즉시 동영상 데이터 다운로드를 시작할지 여부를 브라우저에 알립니다. 옵션은자동 , 메타데이터 , 그리고없음 . HLS 재생 기술의 경우 설정은 약간 다른 것을 의미하며 아래에 설명되어 있습니다.

  • 자동 (기본): 즉시 비디오 로딩을 시작하십시오 (브라우저가 동의하는 경우). iPhone 및 iPad와 같은 일부 모바일 장치는 사용자' 대역폭을 보호하기 위해 비디오를 미리 로드하지 않습니다. 이것이 바로 그 값을 auto라고 부르며 true처럼 더 최종 값이 아닌 이유입니다 .

    HLS 재생 기술의 경우 auto는 플레이어가 브라이트코브 플레이어 버전 5.3 이전의 경우와 같이 m3u8 매니페스트 파일과 3-4 대신 하나의 세그먼트만다운로드한다는 것을 의미합니다.

  • 메타 데이터 : 비디오의 지속 시간 및 크기와 같은 정보가 포함된 비디오의 메타 데이터만 로드합니다.

    HLS 재생 기술의 경우, 메타데이터와 같다자동 .

  • 없음 : 비디오 데이터를 미리 로드하지 마십시오. 이 작업은 사용자가 재생을 클릭하여 다운로드를 시작할 때까지 기다립니다. iOS에서는 이 설정을 사용해도 세그먼트 다운로드가 있을 수 있습니다. 자세한 내용은브라이트코브 플레이어 및 iOS문서를 참조하십시오.

    HLS 재생 기술의 경우, 없음사용자가 재생을 누를 때까지 플레이어가 m3u8 매니페스트를 포함하여 더 이상 아무 것도 로드하지 않음을 의미합니다.
      <video-js preload ...>
      or
      { "preload": "auto" }
참고: 서버측 광고 삽입 (SSAI) 을 사용하는 경우 브라이트코브는 설정을 권장합니다preload=none . preload가auto또는metadata로 설정된 경우 비디오의 첫 번째 프레임스트림이 요청되며, 사용자가 재생을 시작하지 않더라도 광고를 게재한 것으로 계산합니다.
