소개

이 문서에 표시된 curl 문은 여러 계정을 반복하고 업데이트만 합니다. 브라이트코브 기본 플레이어해당 계정에서 원하는 Brightcove Player 버전으로. 업데이트 및 게시를 수행하는 데 사용되는 curl 문에 대한 전제 조건으로 curl 문에서 사용할 액세스 토큰을 얻는 방법을 보여줍니다.

액세스 토큰 가져오기

제공하는 액세스 토큰이 있어야 합니다. 플레이어 읽기/쓰기 curl 문을 사용할 모든 계정에 대한 액세스 권한입니다. Studio에서는관리자드롭다운 선택 API 인증 . 새 애플리케이션을 등록하고 원하는 계정을 선택합니다. 를 위해노출된 Brightcove API선택하다플레이어 > 읽기/쓰기 .

저장 후 복사클라이언트 ID그리고클라이언트 비밀 .

이 프로세스에 대한 자세한 내용은액세스 토큰 생성문서.

이제 액세스 토큰을 얻으려면 다음 샘플에서 클라이언트 ID와 클라이언트 암호를 사용하십시오. OAuth API 샘플: 액세스 토큰 받기 . 액세스 토큰은 5분 동안만 유효하므로 이 샘플을 반복해서 사용해야 할 수 있음을 기억하십시오.

컬 문

몇 개의 파일을 만들고 터미널에서 스크립트를 실행할 폴더를 만듭니다. 해당 폴더에 다음과 같은 파일을 생성합니다. 계정 목록.txt , 파일에서 모든계정 ID업데이트하려는브라이트코브 기본 플레이어 .

이제 다음과 같은 다른 파일을 만듭니다. 일괄 업데이트.sh . 파일에 다음 curl 문을 배치합니다.

#!/bin/bash

#Loop through publisher IDs in accountlist.txt
while read -r ACCOUNTID; do
    echo $ACCOUNTID

    #Update
    curl -X PATCH "https://players.api.brightcove.com/v1/accounts/$ACCOUNTID/players/default/configuration" \
    -H "Authorization: Bearer YOUR_ACCESS_TOKEN_HERE" \
    -H 'Content-Type: application/json' \
    -d '{"player":{"template":{"version":"6.34.3"}}}'

    #Publish
    curl -X POST "https://players.api.brightcove.com/v1/accounts/$ACCOUNTID/players/default/publish" \
    -H "Authorization: Bearer YOUR_ACCESS_TOKEN_HERE" \
    -H 'Content-Type: application/json' \
    -d '{"comment": "Updated to v6"}'

    #Wait 1 second then loop
    sleep 1
done <accountList.txt

물론 두 인스턴스를 교체해야 합니다. 귀하의_ACCESS_TOKEN_HERE실제 액세스 토큰으로. 액세스 토큰을 생성한 지 5분이 넘었다면 샘플을 다시 실행하여 새 토큰을 받으세요. 또한 모든 버전의 플레이어를 사용할 수 있습니다. 사용된 버전을 업데이트하기만 하면 됩니다. 6.34.3 .

마지막으로 두 파일이 있는 폴더에서 터미널 인스턴스를 엽니다. 스크립트를 실행하려면 다음을 입력하십시오.

bash ./bulk-update.sh

스크립트는 플레이어 버전을 업데이트하고 플레이어를 게시합니다.
