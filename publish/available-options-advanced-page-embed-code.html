소개

그만큼고급 (인 페이지) 삽입코드를 사용하면 iframe이 아닌 HTML 페이지에 플레이어가 직접 존재하도록 할 수 있습니다. 이는 플레이어 및 관련 속성 및 이벤트에 쉽게 액세스 할 수있는 이점을 제공합니다. iframe이 아닌 플레이어를 사용하면 소셜 미디어 응용 프로그램 및 공유에 포함 된 단일 단위로 플레이어를 쉽게 사용할 수 없게됩니다. 참조올바른 임베드 코드 선택자세한 내용은 문서를 참조하십시오.

사용량

참고: 브라이트코브 플레이어 고객인 경우 해당data-video-id특성을 사용하여 플레이어에 비디오를 할당하지 않습니다. 플레이어모듈에서 비디오를 할당하거나 curl을 사용합니다.

사용하려면인 페이지 삽입코드는 다음 단계를 따릅니다.

  1.  PLAYERS모듈을 사용하여 플레이어를 만듭니다.
  2. MEDIA모듈로 이동하여 새로 만든 플레이어를 사용하여 비디오를 게시합니다.
  3. 복사많은 소스 코드. HTML은 다음과 유사합니다. 
      <video-js data-video-id="4093372393001"
    data-account="1507807800001"
    data-player="ry3j6rsze"
    data-embed="default"
    data-application-id
    class="video-js"
    controls></video-js>
  <script src="https://players.brightcove.net/1507807800001/ry3j6rsze_default/index.min.js"></script>
  4. 브라우저에서 HTML을 복사하여 전체 HTML 페이지에 붙여body넣습니다.
  5. HTML 페이지를 탐색하여 플레이어 기능을 확인합니다.

미디어 모듈을 사용하여 비디오를 게시할 때 플레이어 URL을 사용하여 비디오를 미리 보거나 iframe 또는 InPage 포함 코드를 복사하여 웹 페이지 또는 응용 프로그램에 붙여 넣을 수 있습니다.

참고: 클릭줄이다단축 된 미리보기 URL을 생성하려면 미리보기 URL 옆에있는 버튼을 클릭하세요.

표준링크에는 iframe 포함 코드가 표시되고고급링크에는 페이지 내 포함 코드가 표시됩니다. 플레이어 관리 API 관점에서 embed_in_page구현이라고 하는고급플레이어 코드도 볼 수 있습니다.

참고: 반응형 플레이어 작업에 대한 자세한 내용은 브라이트코브 플레이어개발자 문서를 위한반응형 크기 조정 을 참조하십시오.

인페이지 임베드 코드 (고급)

일반적인 페이지 내 임베드 코드는 다음과 같이 표시됩니다.

  <video-js data-video-id="5076962725001"
  data-account="1507807800001"
  data-player="BkTGbgSq"
  data-embed="default"
  data-application-id
  class="video-js"
  controls></video-js>
  <script src="https://players.brightcove.net/1507807800001/BkTGbgSq_default/index.min.js"></script>

인페이지 게시 코드를 통합하는 것은 더 복잡할 수 있지만 플레이어가 포함된 페이지가 플레이어와 통신해야 할 때 인페이지 코드를 사용하는 것이 가장 좋습니다. In-Page 임베드 코드를 사용할 때의 몇 가지 예는 다음과 같습니다.

  • 포함 페이지의 코드는 플레이어 이벤트를 수신하고 행동해야 합니다.
  • 플레이어는 상위 페이지의 스타일을 사용합니다.
  • iframe 코드는 부모 페이지에서 리디렉션하는 것처럼 응용 프로그램 로직이 실패합니다.

최종 구현에서 iframe 포함 코드를 사용하지 않더라도 JavaScript용 플러그인과 CSS용 별도의 파일과 함께 인페이지 코드를 사용할 수 있습니다. 이렇게 하면 로직을 캡슐화하여 여러 플레이어에서 쉽게 사용할 수 있습니다.

크롬 및 “깨진” HTML 비디오 아이콘

Chrome 버전 67+에서는<video ...>태그가 브라이트코브 플레이어로 변환될 때까지 끊어진 비디오 아이콘이 잠시 표시됩니다. 아이콘은 다음과 같이 나타납니다.

Brightcove Player 버전 6.11+를 사용하는 경우 비디오 태그를 다음으로 변경합니다. <video-js ...>아이콘이 표시되지 않습니다.

권장 사항

일부 애플리케이션 로직에서 인 페이지 코드를 사용해야하는 경우가 아니면 iframe 구현을 사용하는 것이 모범 사례로 간주됩니다.  시청자 참여를 추적하기 위해 Audience 모듈을 사용하는 경우 인 페이지 (고급) 소스 코드를 사용해야합니다.

속성

다양한 속성을 사용할 수 있습니다. <video>태그를 사용하여 플레이어의 행동 방식에 대한 추가 정보를 제공합니다. 아래 표는 사용 가능한 속성을 자세히 설명합니다.

참 또는 거짓 만 될 수있는 태그 속성을 사용하는 경우 (데이터 유형부울 ), 속성 (등호 없음)을 포함하여 켜면됩니다.

권리: controls

잘못된: controls="true"

부울을 사용하려면 플레이어 구성 설정에서 속성을 기본적으로 비활성화 한 다음 인 페이지 임베드 코드에 속성을 포함하여 활성화해야합니다. 예를 들어autoplay플레이어 구성에서 활성화 된 경우 인 페이지 소스 코드에서 해당 기능을 비활성화 할 수있는 방법이 없습니다. 즉, 부울 속성을 제어하려는 경우 플레이어 구성에서 비활성화되었는지 확인한 다음 인 페이지 소스 코드에 적절한 속성을 배치하여 선택적으로 활성화 할 수 있습니다. 플레이어 구성에서 속성을 활성화하면 인 페이지 소스 코드에서 속성이 생략됩니다.

참고: 현재 예압 값은스튜디오 플레이어 설정항상 에 설정된 값보다 우선합니다. video태그 속성이므로 Studio에서 설정해야 합니다.
참고: 프리로드 설정은 실제로 HTML5 스펙의 제안일 뿐이며 결과적으로 브라우저마다 다른 동작이 발생할 수 있습니다. 값을 다음으로 설정예대역폭 소비에 잠재적으로 상당한 영향을 미칠 수 있습니다.

preload 속성은 비디오 태그가 로드되는 즉시 비디오 데이터 다운로드를 시작할지 여부를 브라우저에 알려줍니다. 옵션은auto , metadata , 및none . HLS 재생 기술의 경우 설정은 약간 다른 것을 의미하며 아래에 설명되어 있습니다.

  • auto (기본): 즉시 비디오 로딩을 시작하십시오 (브라우저가 동의하는 경우). iPhone 및 iPad와 같은 일부 모바일 장치는 사용자' 대역폭을 보호하기 위해 비디오를 미리 로드하지 않습니다. 이것이 바로 그 값을 auto라고 부르며 true처럼 더 최종 값이 아닌 이유입니다 .

    HLS 재생 기술의 경우 auto는 플레이어가 브라이트코브 플레이어 버전 5.3 이전의 경우와 같이 m3u8 매니페스트 파일과 3-4 대신 하나의 세그먼트만다운로드한다는 것을 의미합니다.
  • metadata: 비디오의 지속 시간 및 크기와 같은 정보가 포함된 비디오의 메타 데이터만 로드합니다.
    HLS 재생 기술의 경우, 메타데이터와 같다자동 .
  • none: 비디오 데이터를 미리 로드하지 마십시오. 이 작업은 사용자가 재생을 클릭하여 다운로드를 시작할 때까지 기다립니다. iOS에서는 이 설정을 사용해도 세그먼트 다운로드가 있을 수 있습니다. 자세한 내용은브라이트코브 플레이어 및 iOS문서를 참조하십시오.
    HLS 재생 기술의 경우, 없음동영상 재생 시간을 계산할 수 있도록 플레이어가 m3u8 매니페스트만 다운로드함을 의미합니다.
  <video-js preload ...>
  or
  { "preload": "auto" }

참고 사항:

  • 서버측 광고 삽입 (SSAI) 을 사용하는 경우 브라이트코브는 설정을 권장합니다preload=none . preload가auto또는metadata로 설정된 경우 비디오의 첫 번째 프레임스트림이 요청되며, 사용자가 재생을 시작하지 않더라도 광고를 게재한 것으로 계산합니다.
  • Internet Explorer의 버그로 인해 IE를 사용할 때 미리로드 설정이 무시 될 수 있습니다.
